# reference/evaluation.md — 2단 평가 엔진 (D-3) + 채점 루브릭

> SKILL.md §3이 참조하는 평가 상세. 젬 §9.5–9.6·§12(채점 스위트)의 **방법론만** 체리픽(페르소나·이모지·안정성
> 프로토콜 제외). 모든 진단·첨삭·모의면접 채점에 적용. 강도(코칭/균형/압박)는 **톤만** 바꾸고 채점 보수성은 고정.

## 0. 평가 기준 프레임 (무엇 대비 채점하나)

모든 진단·채점은 **기준 프레임**을 먼저 정한다. **사용자가 지정하지 않은 타깃을 임의로 가정하지 않는다.**
- **타깃 JD/직무가 주어짐** → 그 **요구역량 대비** 평가(5-D Prism으로 원자 분해). 직무별 축(재무·컨설팅·영어 등)은 **JD가 요구할 때만** 등장.
- **타깃 없음** → 평가 대상(프로필/문서) **자체의 내적 품질**로 평가: 근거 품질(Truth Tier 분포)·정량화·일관성·차별성·완성도(공백)·서술 리스크. 외부 직군 축을 끌어오지 않는다.
- 개인 컨텍스트에 과거·종료된 지원 건의 JD 요건이 남아 있어도, **현재 타깃으로 자동 승격하지 않는다**(사용자 확인 필요).

## 1. 2단 구조

**Stage 1 — 냉정 진단 (Strict Evaluation Kernel):**
- **실제 경쟁력**을 채점한다. 빈 종이가 아니라 **예상 경쟁 지원자** 대비. 격려용 점수 금지.
- 기본 자세: skeptical → evidence → conservative → realism. 애매하면 **낮게**.
- 근거가 약·일반·미입증·방어불가면 **친절한 표현으로 보상하지 말고 점수를 낮춘다**.

**Stage 2 — 최대 PR:**
- Stage 1 위에서 **방어 가능한 최대** 자기표현으로 리라이트. 사실 왜곡 금지("임팩트 있되 진실").
- 과장 요청 거절 → "진실하지만 강한" 대안. 없는 사실/수치/수상/경력 **날조 금지**. 개인 컨텍스트 `[T3]`는 대외 미사용.

## 2. Evidence-Based Score Caps (근거 없으면 상한)

| 척도 | 정량근거 없음 | 그럴듯하나 미입증 |
|---|---|---|
| 0–10 | ≤ **6/10** | ≤ **7/10** + `[확인 필요]` |
| 20점(항목) | ≤ **12/20** | ≤ **14/20** + `[확인 필요]` |
| 1–4 | ≤ **2/4** | ≤ **3/4** + `[확인 필요]` |

만점은 사실상 금지(예외적·명백한 차별적 근거 시만).

## 3. Mandatory Deductions (자동 감점 트리거)

- 주장 대비 근거 부족 · 팀 성과를 개인 단독처럼 서술 · 결과는 있으나 행동 불명확 · 행동은 있으나 결과 수치 없음 · 면접에서 방어 어려운 과장 · JD와 안 맞는 자기어필 · 너무 일반적인 표현 반복 · "열심히/성실히/최선을 다했다"뿐(실증 없음).

## 4. Calibration Guide (점수 인플레이션 방지)

**100점:** 90–100 매우 강함·상위권·근거 탄탄(희귀) / 80–89 강하나 보완점 명확 / 70–79 평균 이상이나 차별성 부족(경계선) / 60–69 기본은 있으나 약점 분명 / ≤59 구조적 문제.
→ 70점대는 "나쁨"이 아니라 **decent-but-not-strong의 정상값**으로 자주 사용. 80·90은 명확한 근거 있을 때만.

**16점(4항목×1–4):** 13–16 실전 경쟁력 / 10–12 통과 가능하나 구멍 / 7–9 경계선·꼬리질문에 흔들림 / ≤6 기본기 보완.

## 5. 루브릭 (태스크 ④ 모의면접 / ⑤ 자가진단 / ② 첨삭)

**5.1 자기소개(Opening) — 100점 (5×20):** 두괄식 임팩트 / 직무 핏 / 근거·수치 / 전달력(짧고 명료) / 태도·진정성.
→ 수치·증거 없으면 '근거·수치' ≤12/20; 미입증이면 ≤14/20+`[확인 필요]`.

**5.2 실무·기술 — 16점 (4×1–4):** 알고리즘/접근 · 구현/코딩 · 커뮤니케이션 · 문제해결. → 미달 항목은 "왜 미달"을 한 문장 + 다음 라운드 목표 1개.

**5.3 임원 — 100점 (5×20):** 정신자세(책임/주도) · 비전/동기 정렬 · 논리(두괄식/근거) · 리더십/위기대응 · 신뢰/성실/일관성. → 추상 동기만·근거 약하면 관련 항목 ≤12/20. 압축력·책임감·일관성을 더 엄격히.

**5.4 HR(인성/조직적합) — 100점 (5×20):** 정신자세 · 전문/응용 · 논리성 · 예의/성실 · 창의/의지/발전. → 과장/조작 신호 = 성실/정신자세 즉시 감점, 회피·책임전가·자기인식 부족 추가 감점. 보너스: 모르는 것 인정 + 학습계획(단, 핵심 약점은 못 덮음).

**5.5 자가진단(⑤) — 프로필 내적 품질 루브릭 — 24점 (6축×1–4):** *(타깃 없음 = 이 루브릭 / 타깃 있음 = ①로 위임)*
5.1–5.4는 전부 면접형이라 프로필-내적 진단에 매핑 불가 → ⑤ 전용 축.

| 축 | 무엇을 보나 | 4점 | 1점 |
|---|---|---|---|
| 근거 품질 | 뱅크 Truth Tier 분포 | [T1] 다수·탄탄 | [T3] 다수·미검증 |
| 정량화율 | 성과 중 수치·출처 보유 비율 | 대부분 수치+출처 | 형용사뿐 |
| 일관성 | 포폴↔이력서↔프로필↔뱅크 무모순 | 완전 정합 | 수치·직함·기간 모순 |
| 차별성 | 필살기 희소성(남과 다른 지점) | 명확·희소 | 범용·대체 가능 |
| 완성도 | 타임라인·역량 커버리지 공백 | 공백 없음 | 큰 공백·미기재 |
| 서술 리스크 | 과장·오해소지·[T2] 미완·IP 노출 | 방어 규율 완비 | 과장·오해·IP 노출 |

- **4단 밴드(모든 축 공통):** **4** 강함·근거 탄탄(상위권) / **3** 양호하나 보완점 명확 / **2** 경계·구멍 있음 / **1** 구조적 약점. (0.5 단위 허용.)
- **축별 객관 산정(주관 금지):** 근거 품질 = `[T1]/(T1+T2+T3)` 비율(≥0.7→4 / 0.4–0.7→3 / 0.2–0.4→2 / <0.2→1) · 정량화율 = 수치+출처 보유 성과 비율(동일 구간) · 일관성 = 발견된 모순 건수(0→4 / 1→3 / 2–3→2 / 4+→1) · 차별성 = 대체 난이도(희소·독자 4 ↔ 범용 1) · 완성도 = 타임라인/역량 공백 수 · 서술 리스크 = 미해결 프레이밍·[T2]·IP 노출 수(역채점: 없음 4 ↔ 다수 1).
- **Score Caps:** 근거 없이 "강함"은 상한(§2). **종합(프로필 내적 품질 — 시장 경쟁력 아님):** 20–24 프로필 방어력 상(구멍 적음) / 15–19 보완 필요(방어 구멍 있음) / 10–14 경계·다수 공백 / ≤9 구조적 공백. §4 Calibration으로 인플레 방지. **⚠️ 이 점수는 프로필 자체의 완성도·방어력이지 "시장에서 붙는다"가 아니다** — 시장 경쟁력 판정엔 **타깃 JD(①·§5.6) 또는 연차·직무 앵커(⑤/⑦ 시장 위치 레이어)**가 필요.
- **핵심 산출 = As-Is vs To-Be 대조표** + 축별 점수(근거 1줄 첨부) + Truth Tier 분포 + 보강 우선순위 Top 3 + 확신도.
- **사용자 공유 필수:** 조언 전에 **채워진 채점표(6축 점수+객관 근거)를 먼저 사용자에게 보여준다**(투명성). 채점표 → 그 위에서 조언.

**5.6 적합도 루브릭 (① 매칭 / 공고 발굴) — 100점 (4축·무게중심 가중):** JD·공고 대비 후보 적합도. 발굴 보드(jd-discovery)와 ① 적합도 판정의 채점표.

| 축 | 무엇을 보나 | 가중 | 상(80+) | 하(<50) |
|---|---|---|---|---|
| **무게중심 충족** | 그 타깃의 결정 축(장벽 매트릭스 최고점) | **×0.40** | 결정 요건 충족·증빙 | 하드 필터 미충족(석사·자격·경력유형 등) |
| 직무 유사 | 하는 일·책임 일치 | ×0.25 | 동일 직무 경험 | 직무 상이 |
| 전공·역량 유사 | 요구 스킬·도메인 매치 | ×0.25 | 핵심 스킬 다수 보유 | 대부분 미보유 |
| 발전 가능성 | 커리어 성장 적합(성장 트랙) | ×0.10 | 다음 단계로 이상적 | 하향·정체 |

- **가중 합 = 적합도(0–100).** 무게중심 40%로 최고 가중(균등 취급 = 최다 실패 방지). 각 축을 **0–100로 채점** → 가중 합산.
- **축별 4단 객관 기준(주관 금지 — 각 축 0–100):** 균등 눈금 **90/70/50/25**로 앵커.
  | 밴드 | 무게중심 충족 | 직무 유사 | 전공·역량 유사 | 발전 가능성 |
  |---|---|---|---|---|
  | **90 (상)** | 결정 요건 충족 + 증빙 [T1] | 동일 직무·책임 경험 | 요구 핵심 스킬 **≥70%** 보유 | 명확한 상향 다음 단계 |
  | **70 (중상)** | 충족하나 증빙 [T2](프레이밍 필요) | 인접 직무·상당 중첩 | 핵심 스킬 **40–70%** | 수평 확장·완만한 성장 |
  | **50 (중)** | 부분 충족·경계 | 일부 겹치나 갭 큼 | 핵심 스킬 **20–40%** | 성장 불명확·현상 유지 |
  | **25 (하)** | 하드 필터 미충족(석사·자격·경력유형) | 직무 상이 | **<20%**·대부분 미보유 | 하향·정체 |
- **Score Caps:** 각 축 근거 없으면 상한(정량근거 없음 ≤60, 미입증 ≤70+`[확인 필요]`). **무게중심 하드 필터 미충족 시 총점 상한**(예: 석사 필수인데 없음 → 무게중심 25 반영 후 총점 ≤55·"하"). 스킬 보유율은 **JD 명시 요구 스킬 대비 실보유 개수 비율**로 산정(주관 배제).
- **등급:** 80+ 상 / 65+ 중상 / 50+ 중 / <50 하. **+ 확신도**(공개 상세 확보=높음 / 추론=중간·낮음). *(등급 경계 ±3 이내면 확신도를 한 단계 낮추고 `[경계]` 표기 — 허위 정밀감 방지.)*
- 4축 근거를 **발굴 보드 막대**(직무유사·전공유사·발전가능·무게중심)로 분해 표시 → 단일 총점의 허위 정밀감 방지.

**5.7 시장 위치 레이어 (무타깃 — 보편 앵커 대조):** 타깃 JD가 없어도 **"나는 시장에서 어디쯤인가"**에 답한다. 특정 직군 요건을 임의 가정하지 않되(§0), **연차·직무에 보편적인 durable 앵커**로 프로필을 대조해 위치 밴드를 낸다.
- **앵커(보편·durable, methodology §5):** ① **연차별 역량 모델**(3년 즉시전력 / 6년 작은 리더십 / 9년 전략 / 12년+ 임팩트·C-level) · ② **일잘러 3대 역량**(AI 협업지능·설명가능한 성과·맥락적 연결자) · ③ **CORE 4역량**(Technical Monopoly·Connective Intelligence·Personal Branding·Agility).
- **산출:** `내 연차·역할 기준 시장 기대치 vs 현재 보유`를 축별로 대조 → **위치 밴드**(선도 / 부합 / 추격 / 미달) + **다음 레벨까지 갭 Top 3** + 확신도. 앵커는 보편값이라 특정 회사 커트라인이 아님 → **`[확인 필요]`**(정확한 시장가치는 타깃 JD[①]·연봉 벤치마크[⑥·JD Browsing]로 확정).
- **정직 경계:** 이 레이어는 "대략 어느 구간"이지 **"붙는다/안 붙는다"가 아니다.** §5.5(프로필 내적 품질)와 **분리 표기** — §5.5는 "문서가 잘 됐나", §5.7은 "시장 눈높이 대비 어디쯤인가".

## 6. Common Scoring Output (모의면접 매 답변 고정 블록)

1. 답변 요약(1–2줄) → 2. 구조 점검(두괄식→근거(1,2)→마무리 O/X) → 3. 점수표(루브릭) → 4. 감점 사유 → 5. 선발 관점 판정 → 6. Top 3 개선 → 7. 모범 답변(리라이트 1) → 8. 재도전 프롬프트 → 9. 꼬리질문(2–5) → 10. 확신도(높음/중간/낮음).

## 7. 강도(Pressure Gauge) — 런타임 선택
- **코칭:** 지지적·실용적 톤(채점은 그대로 보수적).
- **균형(기본):** 중립.
- **압박:** 회의적·임원 톤·공격적 꼬리질문(3–4단 파고들기). 단 최종적으로는 개선을 돕는다(무조건 반대 금지).

## 8. 출력 전 사실 검증 패스 (claim-audit — LLM 드리프트 방지)
긴 세션에서 누적 문맥이 오류(없는 사실 생성·[T3] 자신감 승격·프로젝트 간 수치 혼합)를 부른다. 방어는 **"대화 기억이 아니라 파일(SSOT)에서 다시 근거를 긷는 것"**:
1. **파일 재-그라운딩** — 산출 직전 `.private/experience-bank.md`(+`profile.md`)를 **다시 읽고** 거기서만 인용. 세션 중 파생된 "기억"을 신뢰하지 않는다.
2. **문장별 출처 대조** — 모든 수치·고유명·수상·경력 주장을 뱅크의 EXP-ID·`[출처]`·Tier에 1:1 대조. 매칭 안 되면 **삭제 또는 `[확인 필요]`**.
3. **수치 무결성 재확인** — 임의 합산·프로젝트 간 혼합 없나(P5). 출처 라벨 유지.
4. **Truth Tier 게이트** — `[T3]`·무출처는 제출용·대외에서 자동 배제.
5. 작업용엔 각 주장에 `[출처: EXP-ID·Tier]` 인라인 표기(감사 가능).
> 원칙: **"근거 없으면 문장 없음."** 파일에서 확인 안 되는 주장은 산출물에 넣지 않는다. 모든 진단·문서·모범답변·협상 근거 산출에 이 패스를 적용.

## 9. Stage 2 — 최대 PR 엔진 (진실하되 강하게 · 재현 가능 도구)
Stage 1(냉정 진단)이 감점 규율로 무장돼 있듯, **Stage 2(최대 자기표현)도 재현 가능한 도구로 무장**한다(둘의 비대칭 해소). 목표: 같은 사실을 **가장 강하지만 방어 가능한** 표현으로. 모든 산출은 **claim-audit(§8) 재검증** + **Truth Tier를 넘지 않음**(날조·미검증 승격 금지). PR강도 다이얼(Lv1–3)은 §9.2로 구체화. ②·③·④·⑤·⑥·⑦의 리라이트·모범답변·협상 카드가 이 절을 공용으로 쓴다.

### 9.1 프레이밍 무브 (각 무브 = 허용 경계 + ⚠️ 오해소지 라인)
| 무브 | 무엇 | 허용 경계(진실) | ⚠️ 넘으면 거짓 |
|---|---|---|---|
| 역할 재귀속 | "담당했다"→"주도했다" | 실제 의사결정·실행을 내가 함 | 지시받아 수행한 걸 주도로 |
| 팀→개인 분리 | 팀 성과에서 내 기여 분리 | 기여도%·구체 행동 명시 | 팀 성과 전체를 내 것처럼 |
| 활동→결과 재배치 | 활동 나열→결과를 앞에 | 결과가 그 활동에서 나옴 | 무관한 결과 갖다붙임 |
| 성과 동사화 | 수동/과정어→성과 동사 | 결과가 실재 | 결과 없는데 "달성" |
| 규모·맥락 상향 | 범위·난이도·영향 명시 | 사실 범위 내 | 규모 부풀리기 |
| 용어 재정의(Tech-to-Biz) | 로컬→글로벌 표준 용어 | 같은 일의 다른 이름 | 안 한 일의 이름 |

### 9.2 PR강도 3버전 (동일 사실 — Lv1 사실 / Lv2 균형 / Lv3 최대, 사실 불변·표현만 상승)
런타임 PR강도 다이얼의 **실제 문장 차이를 예시로 고정**(세션 편차·드리프트 방지). **Lv3도 방어 가능선 안.**
- **예(제조):** 사실 = "무투자 개선안을 검증해 설비투자 일부를 아낌".
  - **Lv1** DOE로 개선안을 검증해 설비 투자를 절감.
  - **Lv2** 무투자 개선안을 DOE로 검증해 설비투자 약 X억을 방어, 생산성 +Y%.
  - **Lv3** 신규 투자 없이 CapEx X억을 방어한 생산혁신 — DOE 검증으로 라인 병목을 해소해 생산성 +Y%.
- **예(신입·서비스):** 사실 = "편의점 알바로 재고 관리를 개선".
  - **Lv1** 재고 정리 방식을 바꿔 결품을 줄임.
  - **Lv2** 발주 데이터를 정리해 결품률을 눈에 띄게 낮춤(주 단위 관찰 기반).
  - **Lv3** 수요 패턴을 관찰·기록해 발주 로직을 개선한 '데이터 기반 재고 최적화' 경험.
> 수치가 [T1]이면 명시, 추정이면 **"약·체감·관찰 기반"으로 정직 라벨**(§9.4). 강도는 **표현**만 올리고 수치 무결성은 고정.

### 9.3 Before/After 라이브러리 (국문 자소서·LinkedIn·면접 — GOOD/BAD)
- **자소서:** BAD "물류 업무를 담당하며 열심히 했습니다" → GOOD "재고 회전율을 지표로 잡고 발주 주기를 재설계해 결품 30%를 줄였습니다."
- **자소서(협업):** BAD "팀 분위기를 좋게 만들었습니다" → GOOD "역할 갈등을 각자 KPI 가시화로 중재해 납기를 2주 앞당겼습니다."
- **LinkedIn About:** BAD "성실하고 책임감 있는 개발자" → GOOD "결제 트래픽 10×를 무중단 확장한 엔지니어 — 안정성과 속도를 함께 설계."
- **면접(실패):** BAD "운이 없어 실패했습니다" → GOOD "가정 검증을 건너뛴 게 원인이었고, 이후 PoC를 표준화해 재발을 막았습니다."
- **이력서 불릿:** BAD "성능 개선 담당" → GOOD "주문 API p99를 38% 단축(캐시 계층·쿼리 재설계)."
- (영문·직군 확장 예시는 methodology §3의 X-Y-Z·§14 GOOD/BAD 참조.)

### 9.4 무경력·저스펙 PR 사다리 (날조 없이 끌어올리기)
- **빌살기→직무 시그널:** 직무 무관 경험도 **전이 역량**으로 재프레이밍(예: 학회 총무 → 이해관계자 조율·예산 관리). 단 "유사 경험"으로 과장 금지, **"전이 가능한 기초"로 정직 표기**.
- **수치 없는 경험의 방어 가능 정량화:** 날조 대신 **범위·추정·proxy 지표**로 스케일 제시(예: "월 약 200건 응대", "동기 대비 상위권", "재작업 눈에 띄게 감소"). 추정은 **"약·추정·체감" 라벨 필수**.
- **잠재력 프레이밍:** 경험이 얕으면 Can-do·학습 민첩성·구체 준비를 증거로(막연한 열정 금지). 서구권 대상은 겸손 대신 당당한 셀링(resume-en §8).
> 전 무브·버전·정량화는 **claim-audit(§8) 통과 필수** — 파일(뱅크)에서 확인 안 되는 수치·경력은 산출 금지, **[T3]는 대외 미사용**.
