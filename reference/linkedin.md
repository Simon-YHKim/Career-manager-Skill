# reference/linkedin.md — LinkedIn 최적화 & 반영 (필드 카탈로그 + 두 반영 모드)

> ② 문서 태스크의 LinkedIn 산출을 위한 durable 지식. **뱅크에서 파생**해 LinkedIn 전 입력 포맷을 채우고,
> 사용자가 **선택(활성화)**한 섹션만 반영한다. 산출 = [`templates/linkedin-export.html`](../templates/linkedin-export.html).
> 두 반영 모드: **A. 복사-붙여넣기(기본·안전)** / **B. computer-use 자동 입력(런타임 가능 시·게이트)**.

## 0. 원칙
- **뱅크 SSOT + claim-audit.** 모든 값은 `.private/experience-bank.md`에서 파생(§6.0 #7). `[T3]`·미검증은 대외 프로필에 넣지 않는다(날조 금지).
- **문체 게이트.** About·Experience 등 서술은 [`writing-voice.md`](writing-voice.md) 통과(AI 티 제거·성과 동사). 이모지는 LinkedIn 관례상 About 최소 허용(과용 금지).
- **최대 커버 + 사용자 선택.** LinkedIn이 제공하는 입력 포맷을 **최대한 모두** 준비하되, 실제 반영은 **사용자가 활성화한 섹션만**. 빈 섹션 강요 금지.
- **글자수 상한 준수.** 아래 카탈로그의 상한(LinkedIn 기준·변동 가능)을 넘기지 않게 실시간 카운터로 트림.
- **자격증명 절대 미저장/미전송.** 로그인은 항상 사용자 본인. B 모드도 예외 없음(§3).

## 2. A 모드 — 복사-붙여넣기 (기본·가장 안전)
1. 사용자가 반영할 섹션 **활성화**(체크) → 스킬이 뱅크에서 그 섹션만 파생.
2. `linkedin-export.html`에 섹션별 필드·글자수 카운터·**[복사]** 렌더.
3. 사용자가 LinkedIn에서 해당 섹션 편집 → 붙여넣기 → 저장. (스킬은 자동 저장하지 않음.)
- 장점: 계정·ToS 리스크 0, 어떤 환경에서도 동작. **이 모드가 기본값.**

## 3. B 모드 — computer-use / chrome 자동 입력 (런타임 가능 시에만)
사용자 런타임이 **브라우저 자동화(computer use·chrome use)**를 지원할 때만(예: Claude cowork, Codex 등 그런 도구가 붙은 환경). 없으면 **A 모드로 폴백** — 억지로 시도하지 않는다.

### 3-1. 가능 여부 판정 → 제안
- 세션에 computer-use/브라우저 제어 도구가 **실제로 있는지 먼저 확인**. 있을 때만 "자동 입력도 가능합니다"를 제안. 없으면 언급만 하고 A 모드로.

### 3-2. 안전 프로토콜 (비협상 — 하나라도 불가 시 중단하고 A로)
1. **자격증명 미취급.** 로그인은 **사용자가 직접** 수행. 스킬은 아이디·비밀번호·2FA 코드를 **묻지도·입력하지도·저장하지도·전송하지도 않는다.** 로그인 화면·2FA·보안 확인(캡차 등)에 도달하면 **정지**하고 사용자에게 넘긴다.
2. **본인 계정만.** 사용자 **자신의** 프로필 편집만. 타인 계정·대량 계정·스크래핑·연결요청 자동화·메시지 발송 **금지**(ToS 위반·자동화 오남용).
3. **섹션별 사전 확인(dry-run first).** 각 섹션을 입력하기 **전에** "이 값으로 이 필드를 채웁니다"를 사용자에게 보여주고 **명시 승인** 후 진행. 저장(‘Save’) 클릭도 사용자 승인 후.
4. **읽기→검증→쓰기.** 현재 필드 값을 먼저 읽어 **덮어쓰기 위험**을 알리고(기존 내용 보존/병합 여부 확인), 값 반영 후 화면으로 **결과 검증**.
5. **한 번에 한 섹션·중단 가능.** 순차 진행, 사용자가 언제든 중단. 실패·불확실하면 그 섹션은 A 모드(복사) 안내로 전환.
6. **날조·과장 불가 승계.** 자동 입력도 [T3]·미검증 값은 넣지 않는다(§6.0 #7 그대로).

### 3-3. 실행 흐름
1. 산출 템플릿에서 **[Fill Plan 내보내기]** → 활성화된 섹션의 `필드 → 값` 순서 목록(구조화 텍스트)을 스킬이 받는다.
2. 사용자에게 "브라우저에서 LinkedIn 로그인까지 직접 해주세요"라고 안내 → 로그인 완료 신호 대기.
3. 프로필 편집 페이지로 이동 → **섹션별**로 3-2의 3·4를 반복(확인 → 입력 → 검증 → 저장 확인).
4. 완료 후 반영 결과 요약 + 실패/보류 섹션은 A 모드 안내.

> 요약: **스킬이 지원 가능한 영역은 전부 지원**하되, 자동화는 **사용자 본인·사용자 로그인·섹션별 승인**이라는 안전선 안에서만. 이 선을 못 지키면 자동화하지 않고 복사 모드로 돕는다.
