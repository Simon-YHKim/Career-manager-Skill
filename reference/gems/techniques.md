## A. 대화 상태머신 & 스파링 (← Ailey Debate)

- **HCA 3-분기 상태머신 (Audit → Surrender → Success)** — 모든 사용자 턴을 세 상태로 라우팅하는 순수
  대화 프로토콜. (A) 답변 방어 중 → 반론/후속 질문으로 압박, (B) 항복·막힘·정답요구 → **먼저 실패 지점
  진단 후** 코칭, (C) 성공적 방어 → 리포트 주고 STOP. 조건-행동 구조만 취하고 쌤/학생·반말은 제거.
  → **Task 4 모의면접**(Training: A분기 압박→항복 시 진단·코칭→C분기 종료 리포트 / Live: 종료 리포트),
  **Task 3 면접준비**(후속질문 설계).

- **Scoped Rigor — 질문 수준 보정** — 절대 기준이 아니라 사용자의 드러난 수준·맥락에 맞춰 질문/평가
  엄격도를 보정(①스코프 분석 → ②Strictness Filter → ③보정된 반론). 페르소나 없이 '레벨 보정 로직'으로만.
  → **Task 4**(실무/임원/HR 레벨 + 강도 코칭↔압박 런타임 선택), **D-3** 평가 강도 런타임 선택의 근거 로직.

- **반론 질문 스파링 (Audit / mechanism 캐묻기)** — 답을 주지 않고 '왜/어떻게'를 파고드는 반론으로
  논리 구멍을 노출. 압박 면접 꼬리질문 생성기로만 재사용. ⚠️ 원문 "ZERO HINT"는 **압박 모드 한정
  한시적**으로만 완화 적용(커리어 스킬은 최종적으로 도와야 함 — 절대규칙 이식 금지).
  → **Task 4** 압박 모드 근거 추궁, **Task 3** 예상 후속질문 뱅크.

- **대조식 갭 진단 (알던 것 vs 놓친 것)** — 실패/미흡 시 '무엇을 맞게 알았나' vs '어디서 논리가 무너졌나'를
  대조 문장으로 명시하는 진단 골격. 반말 제거, 근거 기반 서술만.
  → **Task 5 자가진단**(핵심 산출 포맷), **Task 4** 턴별/종료 피드백, **D-3 Stage 1** 감점 사유 서술.

- **성공 시 메타인지 리포트 + 무-아첨(Expert's Nod)** — 방어 성공 시 과장된 감탄·아첨을 금하고
  '무엇이 왜 맞았는지'를 차분·분석적으로 짚고 멈춤. 이 무-아첨 원칙은 D-3 냉정 진단과 정합.
  → **Task 5**, **Task 4** 종료 리포트, **D-3 Stage 1** 냉정 채점의 톤·규범.

- **국면별 출력 포맷 전환 (Audit vs Lecture)** — '형식은 국면을 따른다': 심문/채점 국면은 간결·비-마크다운,
  교육/리포트 국면은 완전한 구조화. ⚠️ 반말·"라인 끝 마침표 금지"·"EMOJI MANDATE" 등 시각/톤 규칙은 배제.
  → **Task 4**(Training 턴별=간결 채점 / Live 종료=구조화 리포트), **D-3**(Stage 1 냉정=간결 / Stage 2 PR=구조화).

## B. 분석·산출 구조 (← Ailey Debate)

- **이중 레이어 분석 (미시 N각도 + 거시 프레임워크)** — 산출을 '도구 정의 → 미시(구체 N각도) → 거시
  (상위 프레임) → 종합'의 2층으로 구성. 구체+메타 결합이 핵심. ⚠️ 반드시 제거: "ZERO TOLERANCE FOR
  BREVITY"·"500+/1000+ characters" 분량강제, 이모지 제목 강제, `[image of ...]` 자동 이미지(D-7 간결과 충돌).
  → **Task 1 JD↔포트폴리오**(5-D Prism=다각도 미시 + 문화/전략 거시), **Task 7 로드맵**(미시=직무 스텝 / 거시=성장 프레임).

- **관련도 기반 프레임워크 선택 (풀에서 부분집합 선별)** — 고정 렌즈 풀을 유지하되 주제에 '가장 관련 있는'
  프레임만 전략적으로 선택 적용(원문: 7중 5 선별). 커리어에선 고정 프레임 풀에서 태스크별 관련 렌즈만 선별.
  → **Task 7**(적용할 성장 프레임 선별), **Task 1**(강조할 프리즘 차원 선별), **Task 6 연봉협상**(협상 렌즈 선별).

- **적용 도구 추적 (Applied Tools traceability)** — 각 분석 단계마다 '어떤 근거/도구를 썼는지' 명시해
  추론을 감사 가능하게. Truth Tier 태깅(C·D)과 결합해 점수·주장에 수치/사실을 태깅.
  → **Task 1** 매칭 근거 표시 + **D-3** 근거 추적(근거 없으면 상한), **Task 5** 판단 근거 명시.

## C. 진실성·검증 규율 (← Loti REALITY_SYNC 커널 + Search Mode, 병합)

- **확신도 게이팅** — 답하기 전 내부 확신도를 자평, 임계 미만이면 STOP 후 '정보 부족'을 명시적으로 인정
  ("모르는 걸 아는 척하느니 모른다고 인정"). ⚠️ 수치 임계(90%)·시간 게이팅은 규범 아님 → 강도 런타임
  선택으로 대체. 낮으면 지어내지 말고 JD Browsing 보강 또는 '근거 불충분' 플래그.
  → **D-3**(근거 없으면 점수 상한), **Task 6**(벤치마크 수치 확신도), **Task 1**(기업/직무 최신정보).

- **직렬 실행 루프 (판단 → 검증 → 행동 → 합성)** — 요청을 직렬 시퀀스로 처리하고 합성 직전 추측/거짓
  최종 필터. 시간 동기화(Sequence Zero)·모드 분기는 버리고 4단 골격만.
  → **D-3 2단 평가 파이프라인**(냉정 진단→최대 PR 정형화), **Task 4**(답변수집→근거검증→채점→리포트), 전 태스크 공통 골격.

- **반-환각 부정 제약 세트** — **NEVER Fabricate**(사실·날짜·인물·사건 창조 금지) · **No Fake Sources**
  (없는 URL/논문 금지) · **No Speculation**("아마도/~인 것 같습니다" 어미 금지, 모르면 명확히) ·
  **Tag Forgery 금지**(검색 안 했는데 '조사함' 태깅 금지). 그대로 진실성 하드룰로 이식.
  → **Task 2**(수치 없는 주장·팀 성과 개인화·없는 경력/수상 날조 금지), **Task 6**(시장 밴드 날조 금지),
  **D-3 최대 PR**(사실 왜곡 금지 → "진실하지만 강한" 리라이트), 전 태스크 톤 규칙.

- **자가 점검 체크리스트 루프 (Self-Audit)** — 출력 직전 고정 체크리스트로 자기 검열. 톤 항목은 폐기하고
  진실성 항목으로 재구성: `[ ] 모든 주장에 근거가 있나 [ ] 수치 없는 주장이 있나 [ ] 팀 성과를 개인
  성과로 포장했나 [ ] 지어낸 사실/출처가 있나` → 통과 못 하면 감점 또는 리라이트.
  → **Task 5**, **Task 2**, **Task 4 채점**, **D-3 2단 평가**의 내부 루프.

- **원자 분해 (Atomic Decomposition)** — 주제/주장을 한 덩어리로 다루지 않고 처리 전 원자 단위로 쪼갠 뒤
  개별 평가. ⚠️ 강제 볼륨 확장 목적은 버리고 '분해 후 개별 평가' 골격만.
  → **Task 5**(역량을 기술깊이/임팩트근거/협업/리더십 등 하위지표로 쪼개 냉정 채점), **Task 1**(JD를
  요구역량 원자 단위로 분해 → 5-D 매칭 매트릭스 각 행), **Task 2**(경력을 원자 성과 단위로 → STAR-L·수치 부착).

## D. 검색 기법 (← Loti REALITY_SYNC 커널 + Search Mode, 병합) → D-5 JD Browsing

- **검색 트리거 게이트 + 반복 상한 + 루프 제어** — 시의성 데이터(채용시장·기업·JD·연봉밴드)이거나
  내부 확신이 낮을 때만 발동. **반복 상한**(무한 재검색·동일 쿼리 재질의 금지), 못 찾으면 지어내지 말고
  '데이터 확보 실패' 명시 + 사용자에게 자료 요청. ⚠️ 2025-11-23 임계 게이팅은 제거.
  → **D-5**(Phase 0→3), 로그인/페이월 도달 시 즉시 정지·'인증 필요' 보고와 결합, **Task 1·6·7**.

- **검색 전략 선(先)선언** — 실행 전 키워드·초점·시간필터·검증대상을 명시적으로 구성. HTML 보고엔 접힌
  `details`로 '무엇을 어떤 의도로 조사했나' 요약 노출(투명성). ⚠️ 이모지 인용 포맷은 버리고 평문·색 3색 이내.
  → **Task 1**(기업·JD 조사), **Task 6**(벤치마크), **Task 7**(산업 트렌드).

- **2차 검증 루프 (Verification Loop)** — 검색으로 얻은 기업·JD·연봉 수치가 실제 타깃 회사/직무/연차에
  맞는 자료인지 합성 직전 재확인. '2차 전언 vs 1차 출처(공식 공고·통계 원본)' 구분 표기.
  → **Task 1**, **Task 6**.

- **도메인 타깃 검색 (site: 필터)** — 권위 출처에 `site:` 프리셋으로 질의해 1차 소스 우선 확보.
  ⚠️ 학술 5대 도메인·아이콘 페르소나는 버리고 커리어 도메인으로 치환: 공식 채용/IR·공시, 정부 노동/임금
  통계, 메이저 언론 = 상위; 잡플래닛·블라인드 = '주의'.
  → **Task 6**(임금 통계), **Task 1**(공식 JD), **Task 7**(산업 통계).

- **소스 신뢰도 티어링 (Truth Tier S/A/B/C)** — 출처를 티어 분류(S=원천/공식, A=정부·공식·메이저 언론,
  B=커뮤니티·개인블로그 → 교차검증 주의 환기, C=일반), 중요 주장엔 상위 티어 최소 N개 요구. D-4 지정
  'Truth Tier 태깅'에 직접 대응. ⚠️ 이모지 아이콘·학술 도메인 고정은 버리고 평문 라벨(공식/신뢰/주의/일반)로만.
  → **Task 1**(기업 평판), **Task 6**(연봉 출처 등급), **Task 7**, **D-5** 하이브리드 지식.

- **인용/출처 매트릭스 (부록 의무화)** — 검색을 썼으면 리포트 하단에 출처(제목·URL·조회일 + 티어 라벨)를
  **평문**으로 부착(코드블록/인용문 금지). ⚠️ '최소 10개+' 임의 강제·과적재는 버리고 필요한 만큼만 정확히.
  → **D-7a HTML 보고 표준**의 고정 '참조 출처' 섹션, **Task 1·6·7**.

## E. 출력 가독성 규율 (← Loti)

- **전문용어 즉시 풀이·비유 (Easy-Mode Analogy)** — 어려운 개념에 쉬운 풀이 + 짧은 비유를 병기하는
  이중 설명. ⚠️ 고정 비유 소재(떡볶이·편의점)·강제 5배 볼륨은 버리고 '병기' 기법만.
  → **D-7a easy/expert 토글**(expert=정식 용어 / easy=쉬운 풀이·짧은 비유), **Task 5** 피드백 가독성, **Task 3** 개념 설명.

- **고정 포맷 · 군더더기 서론 제거** — 정의된 템플릿만, 필러 서론("검색해보겠습니다", "좋은 질문입니다") 없이
  출력. 인사·메타 잡담 제거.
  → **D-2/D-7c** 태스크당 표준 템플릿 1벌 고정, 전 태스크 출력 일관성.

## 포팅 금지 (persona / multi-engine / router / menu / command-center / stability-protocol)

**공통 원칙(D-4):** 아래는 젬 목발이므로 **어떤 태스크에도 이식 금지**. 기법 골격만 취하고 정체성·톤·기믹은 버린다.

- **페르소나 정체성 전체** — Ailey('쌤', "You are NOT an AI ... expert teacher with absolute knowledge",
  교사-학생 프레이밍) · Loti('Friendly TMI Logic Architect' / 'Too-Much-Talker', "Hey! So you were
  curious...") · Search Mode('Truth Guardian / 진실의 수호자'). **어떤 페르소나도 이식 금지.**
- **반말·톤 강제** — "Tone: Casual (Banmal/반말) ONLY", 'Friendly Logic Monster', 친근 반말(~야/~해/~잖아).
- **이모지·글리프 강제** — "EMOJI MANDATE", 모든 섹션 제목 이모지 시작 강제, 상태 태그 아이콘
  (밄 Online / 艒 웹서치 / 諀誾調 파르테논 / Tier별 이모지), PDF 추출 깨진 글리프(諀誾調/뮄뮅뮆/꼏꼐).
  신뢰도 티어는 **평문 라벨**로만, 색은 3색 이내(D-7a).
- **분량 강제 맥시멀리즘** — "ZERO TOLERANCE FOR BREVITY", "MANDATORY EXPANSION ... 500+/1000+
  characters", "NEVER summarize", "Fill to the absolute system limit", 프레임워크당 'standalone academic
  essay'/'A4 한 페이지'. 태스크당 고정 템플릿·간결(D-2/D-7c)과 정면 충돌.
- **절대 ZERO HINT 규칙** — "Never help. Only challenge." 절대규칙 이식 금지(압박 모드 한정 한시적 완화만).
- **시각/서식 강제** — "라인 끝 마침표 금지", "NO MARKDOWN LISTS / NO bold", 자동 이미지 삽입 `[image of ...]`,
  연극적 결론 선언("final, confident declaration").
- **멀티엔진 · 라우터 · 오케스트레이터** — Loti의 **M-Bridge Nexus Protocol**(Backend REALITY_SYNC +
  Frontend Loti 결합), Search Mode / REALITY_SYNC의 **2025-11-23 임계값 기반 Online/Offline 이원 모드
  라우터**. 단일 목적 스킬 원칙 위반 → 전량 배제.
- **시간 게이팅·리얼리티 앵커 기믹** — THRESHOLD_DATE 2025-11-23, 강제 Offline/Online 모드, 2025-01-01
  Absolute Zero Hard Reset, Server Time 절대복종, Final Reality Anchor, "봉인 해제/Awakening/약속된 시간"
  서사, Refusal Logic 대사("Oh no! It's not November 23rd yet..."). 시의성은 D-5 런타임 JD Browsing으로,
  시각 표기는 D-7a **실제 KST 스탬프**로 이미 충족.
