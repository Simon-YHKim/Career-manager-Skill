# reference/jd-browsing.md — JD Browsing (내장 웹 리서치 모듈)

> 스킬 **내장·자체완결** 런타임 웹 리서치 모듈. 구 'insane-search' 개념을 **우리 스킬로 클론·내장**
> (외부 스킬 의존 없음). 취업 관련 시의성 데이터를 요청 시점에 공개 웹에서 수집:
> **공고 발굴(JD Discovery)** + 기업·연봉·인재상·JD 원문·직무 트렌드.
> SKILL §4·§6(①③④⑥⑦)·portfolio-builder P0·P3이 참조. Durable 규칙은 `methodology.md`, 여기선 시의성만.

## 0. 원칙 (safety-first)
- **공개 콘텐츠만.** 로그인·페이월 **우회 금지**. **자격증명 저장·전송 금지.**
- **날조 금지.** 실제 검색 결과만. 못 찾으면 '데이터 확보 실패' 명시 + 사용자에 요청.
- **요청일 기준 최신.** 모든 항목에 **출처 + 조회일 + Truth Tier**(S 공식/원천 · A 정부·메이저 · B 커뮤니티 주의 · C 추론·미확인).
- **Phase 0→3**: 0 정의·키워드 → 1 공개 검색 → 2 교차검증 → 3 종합.

## 1. 게이트된 콘텐츠 처리 (로그인/페이월)
채용공고는 **불특정 다수 대상**이라 상세가 로그인 뒤여도 공개 신호는 대개 존재한다. **정지하되 버리지 않는다:**
1. **정지**(우회 안 함) → 어떤 보드/페이지가 게이트됐는지 기록.
2. **공개 신호로 추론**: 제목·회사·애그리게이터 스니펫·검색 캐시·회사 커리어페이지·채용 보도로 "어떤 공고인지" 요약.
3. **`[상세 미확인 — 직접 접속 확인]`** 플래그 부착 + 정확한 요건·마감은 **사용자가 링크를 열어 확인/붙여넣기**하도록 안내.
4. 추론 결과는 Truth Tier로 등급화(추론 = B/C), **확정 아님**을 명시. 사용자 확인 시 [T1] 승격.

## 2. 공고 발굴 (JD Discovery) — 태스크 ① 발굴 모드
**(1) 탐색 프로파일** (뱅크·프로필에서 자동 생성)
- 핵심 역량·필살기 카테고리 · 타깃 방향(프로필 커리어 방향) · 연차/레벨 · 지역/이동성
- **언어 능력 → 공고 언어 범위**: 언어 근거(자격·경험) 있으면 해당 언어권 공고도 탐색(예: 영어 능력 → 국내 + 영어권·글로벌). 근거 없으면 국내 위주.

**(2) 탐색 대상** (요청일 기준, 공개)
- 국내: 원티드·사람인·잡코리아·리멤버·링크드인 KR·회사 커리어페이지
- 해외/글로벌: LinkedIn·Indeed·회사 커리어페이지·도메인 특화 보드
- 직무/도메인 특화 보드는 런타임 판단으로 추가.

**(3) 산출 = 적합도 포트폴리오 매트릭스** (report.html)
`공고/직무 | 회사 | 출처(조회일·Tier) | 요구 핵심(추론 시 플래그) | 내 적합도 등급 | 무게중심 축 | Winning Angle | 추천도 | 지원현실(마감·언어·근무형태)`
- 적합도 등급 = §6.0 2단 평가 + 무게중심(등급 상/중상/중/하 + 확신도). 근거 약하면 Score Caps.
- 정렬: 추천도·마감 임박순. 마감 지난 건 표시.

**(4) 핸드오프**: 선택 공고 → ① 심층 매칭(5-D Prism + 매칭 매트릭스). 관심 공고 → `.private/applications/`에 '관심' 단계로 저장(중복 방지, §6.0 체이닝).

## 3. 기타 런타임 수집 (태스크별)
- **③④ 면접**: 기업 인재상·핵심가치·리더십원칙·전형 단계·현직자 후기(공개).
- **⑥ 연봉**: 시장 밴드·직무 보상 레인지. **정직한 천장**: levels.fyi·블라인드·원티드인사이트 등 고신뢰 comp는 로그인월(→ §1 추론 + 사용자 확인), 공개 통계·기사 위주로 Tier 낮춤.
- **⑦ 로드맵**: 산업·직무 트렌드·미래역량 수요.
- **①/②**: JD 원문·기업 최신정보(7-Lens 입력).

## 4. 인용·표기
- 리포트 하단 '참조 출처'에 평문으로: 출처명 · Tier · 조회일.
- 추론/미확인은 **반드시 플래그**. 확정 요건은 사용자 확인 후 [T1] 승격.

## 5. 커뮤니티 시그널 (현직자·경험자 후기 — 저신뢰 참고)
공식 정보(S/A)가 말하지 않는 **적나라한 현실**(실제 문화·워라밸·연봉 현실·전형 난이도·조직 리스크)을 **참고용으로 병행** 제시한다. 신뢰도는 낮지만 방향 판단에 도움이 된다.

- **소스**: 블라인드·잡플래닛·Glassdoor·Reddit(회사/산업 서브레딧)·직무 커뮤니티(OKKY·긱뉴스 등)·유튜브 후기. 공개 범위만(로그인월은 §1 추론 + `[직접 확인]` 플래그; 블라인드는 대개 게이트).
- **Tier 고정 = B(커뮤니티 주의) / C(단일·미검증)**. **S/A 승격 금지 · 사실로 인용 금지.**
- **표기**: 공식 정보와 **시각적으로 분리된 "커뮤니티 시그널" 블록** + 명시 면책 — *"개인 경험·편향·과장 가능, 사실 아님, 방향 참고용."*
- **노이즈 필터**: **반복 테마(독립 3+ 소스) = 신호 / 단발 불만 = 노이즈**로 구분. **불만 편향**(퇴사자·화난 사람 과대표집) 명시. 대략의 표본 규모·시점 표기.
- **작업용 전용**: 제출용·대외 산출물엔 **미포함**(§6.0 계약 #5·#7). 커뮤니티 주장을 산출물의 사실 근거로 쓰지 않는다.
- **활용처**: ① 히든배리어·근무현실 / ③④ 전형 난이도·면접 후기·실제 조직문화 / ⑥ 연봉 현실·협상 여지.
