## §0. 핵심 프레임워크 (D-4 체리픽 4종) — 캐논

### (1) 5-D Prism — JD↔지원자 5차원 매칭 렌즈
하나의 JD·지원자를 아래 5개 차원으로 분해해 매칭 매트릭스를 만든다(Task 1의 뼈대, Task 2/5에도 사용).
1. **역량(Competency)** — JD가 요구하는 하드/소프트 역량을 원자 단위로 분해, 지원자 경험과 1:1 대응.
2. **의도(Intent / Why-Me·Why-Us)** — 기업의 채용 의도·페인포인트와 지원자 지원동기의 정렬. "왜 경쟁사가 아니라 이 회사인가".
3. **KPI(정량 성과)** — 각 매칭 주장을 수치(%·$·건수·인원·기간)로 뒷받침. 수치 없으면 채점 상한.
4. **공격포인트(Attack Point)** — 냉정한 면접관/스크리너가 파고들 약점·의심(과장, 팀성과 개인화, 도메인 갭, 경력 애매성). 선제 방어 논리 준비.
5. **문화(Culture Fit / Add)** — 조직 가치·일하는 방식과의 적합(Fit)을 넘어 고유 관점의 기여(Add).
→ **산출: 매칭 매트릭스** `요구역량(원자) | 지원자 근거 | KPI | Truth Tier | 갭·보강`.

### (2) Truth Tier 태깅 — 진실성·근거 등급 (D-3 2단 평가의 축)
모든 주장에 근거 등급을 태깅해 "냉정 진단(상한 규율)"과 "최대 PR(왜곡 금지)"을 동시에 지탱.
- **주장 티어:** `[T1]` 문서화·정량 입증됨 · `[T2]` 사실이나 프레이밍/맥락 필요(예: CapEx=투자 방어, OEM=고객사) · `[T3]` 미검증·희망사항 → 검증 전 사용 금지.
- **출처 티어(외부 데이터, gems D 연동):** `S` 원천/공식 · `A` 정부·공식·메이저 언론 · `B` 커뮤니티·블로그(교차검증 주의) · `C` 일반. 중요 주장엔 상위 티어 최소 2개.
- **규율:** 근거 없으면 점수 상한(Stage 1). 과장 요청은 거절하고 "진실하지만 강한" 리라이트(Stage 2). 날조(사실·날짜·수상·출처) 절대 금지.

### (3) STAR-L — 경험 서술 엔진 (문서·면접 공통)
`Situation(2문장, 건조한 객관) → Task(과제 본질 What+Why) → Action(★'나'가 주도, 데이터·툴·방법론 구체) → Result(★정량화, before→after) → Learning(교훈·이후 적용)`.
- 시간/분량 배분: S·T 짧게(10~15%) → **Action 최대(50~60%)** → Result(20~25%, 수치 필수) → Learning.
- '우리 팀' 뭉뚱그림 금지, 수동태·"배웠다" 지양(능동 성과동사). 실패 경험도 Learning까지 완결하면 강력.
- 확장형: 진화된 STAR = S-T-A-R + Insight(재현 가능한 자기 법칙) + Reflection(아쉬운 점).

### (4) Tech-to-Biz — 기술·현장 언어를 비즈니스·재무 성과로 변환
기술·운영 활동을 경영진·채용담당자가 인식하는 가치로 번역(Task 1/2/6 핵심, 특히 제조·엔지니어 직무).
- **환산 공식:** `[기술 활동] → [정량 지표 개선 %/초/수율] → [재무·생산 성과 $/CapEx/Capa%]`. (예: 사이클타임 15%↓ → 연 CAPEX $2M 절감 + Capa 10%↑)
- **용어 재정의(Reframing):** 로컬/운영 용어를 글로벌 표준 전략 용어로 치환(생산기술/공정관리 → Process Optimization·Cost Engineering·Supplier Industrialization·TPM). 직무를 '운영'이 아닌 '전략'으로 재인식시킨다.
- **역할·기여도(%)** 명시 + Truth Tier로 방어 가능성 확보.

## §1 jd-matching — JD↔역량 매칭 방법론

목표 직무의 **채점 무게중심을 먼저 찾고**, 지원자 경험을 그 무게중심에 1:1로 증명·정렬하는 것이 매칭의 본질. 모든 직무를 같은 방식으로 준비하는 것이 가장 흔한 실패.

### 원칙 (rules)
1. **세그먼트 먼저 분리한다.** 지원 대상을 생태계별(예: 국내 제조 대기업 / 글로벌 빅테크 / 유니콘)로 나누면 인재정의·핵심필터·보상구조가 완전히 다르다. 세그먼트마다 준비 전략을 분리하라. (예시 기업·실태는 런타임 재수집)
2. **가장 높은 장벽에 리소스를 몰아준다.** 균등 준비 금지.
3. **JD 표면이 아니라 '히든 배리어'가 당락을 가른다.** JD에 크게 안 드러나지만 가점을 가르는 숨은 요건(특정 규격·라이브 SQL·본사 영어 라운드·매뉴얼 독해 테스트 등)을 먼저 발굴해 대비.
4. **Pivot이 아니라 Expansion.** 기존 도메인(현장/OT/공정) 역량을 버리지 말고, 순수 SW·순수 IT가 못 가진 차별점으로 재포지셔닝.
5. **용어를 재정의(Reframing)한다.** 로컬/운영 용어를 채용담당자가 인식하는 글로벌 표준 전략 용어로 치환(생산기술/공정관리 → Process Optimization·Cost Engineering·Supplier Industrialization·Technical Program Management). 직무를 '운영'이 아닌 '전략'으로 재인식시켜라.
6. **활동을 재무성과로 환산한다.** '공정 최적화 수행'(X) → '[개선 %/초/수율] → [재무성과 $·CAPEX·Capa %]'(O). 수치 없는 주장은 채점 상한.
7. **근무 현실을 먼저 확인한다.** 3교대·출장 비중·램프업 강도 등을 매칭 전에 파악해 지원 후 미스매치 방지.

### 매칭 절차 (checklist)
1. **세그먼트 판정** — 목표가 어느 생태계인지 분류하고 그 교전수칙을 확인.
2. **채용 장벽 매트릭스 점수화** — 목표 직무를 코딩/알고리즘 · 영어 유창성 · 도메인/산업지식 3축에 각 1(낮음)~5(극도로 높음)로 채점. 최고점 축 = 준비의 무게중심.
3. **히든 배리어 식별** — JD·현직자 후기에서 표면에 안 드러난 결정 요건 발굴.
4. **JD 키워드 추출 → 페인포인트-솔루션 매핑** — 기업의 구체적 페인포인트를 지원자 경험이 어떻게 해결하는지 1:1 논증. 직무별 핵심 키워드(예: Should-costing, APQP/PPAP, MES/PLC, 시스템디자인)와 연결 가능한 자기 사례를 집중 정리.
5. **R&D vs 현장 분기 진단(엔지니어 직무)** — 같은 베이스라도 도구·업무호흡·역량이 갈린다. R&D=창조·최적화(심화역학+3D CAD+CAE해석, 석사 선호), 현장=안정화·효율화(troubleshooting/RCA·SPC 통계분석·협업, 학사 주무대). 성향으로 분기를 먼저 정하고 사례를 배치.
6. **역량 변환 매핑(1 역량 → N 직무)** — 하나의 핵심역량을 산업별 여러 직무로 치환(공정최적화 → Cost Innovation / Solution Consulting / Deployment).
7. **적합도 포트폴리오로 우선순위화** — 여러 직무를 한 표로 비교해 1차 타겟에 리소스 집중.

### 프레임워크·템플릿
- **채용 장벽 매트릭스** — [코딩/알고리즘 · 영어 · 도메인지식] 각 1~5점 → 준비 전략 도출.
- **직무 적합도 포트폴리오 매트릭스** — `직무그룹 | 예시기업 | 적합도(★5) | 핵심 어필(Winning Strategy) | 추천도(최우선/우선/추천/도전)`.
- **직무 카드 템플릿** — `핵심 과업 / 필수 스킬 / 근무 현실 / 보상밴드 / 채용 팁` 5칸을 채워 비교.
- **정량화 성과 템플릿** — `[기술 활동] 통해 [정량 지표 개선] → [재무/생산 성과]` (예: 사이클타임 15%↓ → 연 CAPEX $2M 절감 + Capa 10%↑).
- **페인포인트-솔루션 논증 구조** — 기업 JD 페인포인트 → 지원자 경험 → 해결 논리 1:1.
- **하이브리드 엔지니어 삼각형** — Domain Knowledge × Data 해석 × Digital Tool 활용을 각 경험으로 증명. (제조/엔지니어 직무의 현대적 인재상)
- **JD↔메가트렌드 매핑** — JD 문구를 산업 거시 트렌드 축에 매핑해 자소서 키워드·경험을 정렬. (트렌드 목록 자체는 재수집)
- **클러스터 기반 타겟팅** — '근무지 = 커리어 성격'(HQ·선행R&D=기획력 / 첨단 클러스터=기술력 / 양산벨트=현장력). 지원자 강점·성향을 클러스터 성격에 매칭. (지역 지형은 재수집)

### 평가 기준 (매칭 채점 시 무게중심)
- **매트릭스 최고점 축이 곧 채점 축.** 코딩축 高: 알고리즘 효율(Big O)·시스템 디자인·비효율 솔루션 탈락. 영어축 高: 토익 점수가 아니라 기술을 유창하게 설명·설득·논쟁하는 능력(못하면 기술력 무관 불합격). 도메인축 高: 산업 소자/물성·공정 심화와 데이터 툴 실무.
- **데이터 기반 문제해결 도구 숙련** — Six Sigma, 8D Report(Root Cause→Corrective→Preventive), 파레토, 공정능력분석(Cp/Cpk), SPC(Minitab/JMP/Python). 품질/SQE 직무 핵심.
- **NPI·원가 역량** — APQP/PPAP 주도, Should-costing·BOM 최적화·CAPEX 투자 타당성 기술검증(Cost/Value Eng).
- **정량 성과 제시 여부** — 활동을 $·%·수율로 환산했는가.
- **도메인 지식 + 커뮤니케이션 동시 보유** — 현장(OT) 언어를 IT/비즈니스 언어로 번역·설득(DX Consultant, Presales, FAE).
- **도메인 전문용어의 정확한 사용** — 가동률/BM·PM/Yield/SPC/RCA/공차분석/APQP/8D 등을 직무 맥락에 정확히 이식하면 실무 이해도 시그널.

### 시의성 주의 (런타임 JD Browsing)
프레임·절차·평가기준은 durable이므로 내장한다. 그러나 세그먼트 예시기업, 히든 배리어의 구체 요건, 보상 수치(PS/PI/OPI·TC·RSU), 메가트렌드 목록·비중, 클러스터 지형(팹/라인·투자·조직개편), 특정 기업 JD·채용현황·리더십원칙 문구는 **매년/수시 변동** → 프리즈 금지, 매칭 시점에 재수집. 회사 사례는 방법론 예시로만 쓰고 수치는 고정하지 말 것.

## §2 cover-letter — 자기소개서 작성 방법론 (자소서 = 비즈니스 제안서)

**철칙:** 자소서는 '나의 과거 자랑'이 아니라 **기업의 당면 과제에 내가 어떻게 전략적으로 부합하는지 증명하는 제안서**다. 서류 당락은 스펙이 아니라 *지원 회사 채용 의도에 대한 고민의 깊이*가 가른다. 절대 과장·조작 금지 — 자소서는 면접 질문의 유일한 근거이자 입사 후 평가까지 따라붙는 DB(영구 꼬리표).

### 1. 쓰기 전 인지 세팅
- **제안서 재정의:** 기업이 지금 *결핍한 것*을 먼저 해독하고, 내 역량을 그 결핍에 정렬. 상대의 결핍을 모른 채 쓴 글은 공허한 외침.
- **상호 적합성(소개팅):** 뛰어난 역량도 지금 필요로 하는 직무·문화와 결이 안 맞으면 탈락. Person-Job/Org Fit이 스펙보다 우선.
- **면접 대본 설계:** 잘 쓴 자소서는 면접관이 나에게 유리한 질문을 던지도록 유도. 팩트와 진정성으로만 축조해 면접 압박 검증에서 무너지지 않게 한다.

### 2. STAR-L 재조립 (서술 엔진)
- **S (Situation) — 2문장 압축:** 언제·어디서·어떤 맥락인지 건조한 객관 사실만. 연혁·감정·배경을 3~4문장 늘어놓으면 인지 과부하로 즉각 감점.
- **T (Task):** S와 결합해 과제의 본질(What)과 해결해야 했던 이유(Why)를 정의. 중요성을 외부 요인에 떠넘기지 말 것. (S+T 합쳐 최소 분량, 최대 2~3문장)
- **A (Action) — 승부 구간:** '우리 팀은~' 식 조직 뭉뚱그림 금지. **'나(I)'**가 부여받은 역할·판단 근거·실행 과정을 입체적으로. 이공계/제조/IT는 *어떤 데이터로 원인 분석하고 어떤 툴·방법론으로 변수를 통제했는지* 구체성이 핵심 지표.
- **R (Result) — 정량화 or 학습:** 성과는 반드시 수치화(%·시간·비용, 예 "처리 속도 40% 단축"). 모호한 '매출이 올랐다' 금지. **성과가 없거나 실패해도** 과정 복기 → 무엇을 배웠고(Learning) 한계를 어떻게 보완했는지 후속 조치까지 증명하면 합격 자소서(평범한 성공보다 강한 인상).
- **분량 배분:** 핵심 행동·깨달음을 한 문장으로 끝내지 말 것. 왜 그 판단/단어를 선택했는지 + 내 가치관과 어떻게 연결되는지를 충분히 스토리텔링. 핵심에 지면 인색하면 논리 비약으로 설득력 상실.

### 3. 문항별 서술 공식
- **두괄식 주장 = 소제목:** 모든 문항 첫 문장은 서술이 아닌 강력한 '주장'(예: "데이터 기반 합리적 의사결정"). 이 한 문장이 면접관에게 프레임을 씌운다.
- **지원동기·입사 후 포부 [주장→근거→맺음말]:** ①주장 = '동종 경쟁사가 아니라 왜 우리 회사인가'에 답 — 회사 강점뿐 아니라 취약점·돌파 과제까지 객관 분석하고 내 직무 목표와 정렬 선언. ②근거 = 내 강점·기술·경험이 회사 목표/문제 해결에 어떻게 직접 '기여'하는지 현업 언어로 증명. ③맺음말 = 입사 후 3·5·10년 성장 시나리오로 장기 파트너 증명('열심히 배우겠다'식 수동태 금지).
- **성격 장단점 [동시 제시→근거→극복]:** ①직무에 치명상 없는 단점 + 직무 시너지 장점을 동시에. ②장점 발휘 경험 + 단점으로 어려움 겪은 상황을 솔직히. ③[승부처] 주인공은 단점이 아니라 **극복 과정** — 어떤 습관·시스템으로 끈질기게 개선했는지 상술. 단순 인지에서 멈추면 치명적.
- **성장과정:** 일대기 요약 금지(평가자는 '현재의 나를 지배하는 직업윤리·가치관'만 궁금). 가치관에 폭발적 영향을 준 **단일 이벤트/인물 하나**에 집중 → 가치관 두괄식 선언 → 결정적 에피소드를 STAR-L로 깊이 전개.

### 4. 소재 필터링 알고리즘
- **문항 의도 매핑:** 아무리 훌륭한 성과도 문항의 숨은 의도와 어긋나면 과감히 폐기. 뉘앙스를 분석해 경험 DB를 정확한 위치에 배치.
- **위험극복/도전 = 객관적 장애물 우선:** 부족한 예산·이해관계자의 극심한 반대·데이터 치명적 오차 등 누가 봐도 명백한 물리적·시스템적 한계를 1순위. 심리적 우울·개인 슬럼프 등 주관적·감정적 경험은 배제.
- **팀워크/인성 = 구조적 갈등 해결:** '분위기 좋았다/칭찬받았다'는 피상적 호응 사례 부적합. 역할분담 마찰·희생 수용·소통 부재 등 조직 병목을 내가 어떤 소통·솔선수범으로 뚫었는지 매핑. 진짜 팀워크·리더십은 충돌의 조정·타협으로 증명.

### 5. 감점 방어 (Red Flag 체크리스트)
- ❌ 만연체(장황·S/T 비대) · ❌ 허위·과장 · ❌ 복붙형(회사 커스터마이징 없음) · ❌ 진부한 클리셰.
- 피할 클리셰: 조별과제 갈등, 군대 인내심, 국토대장정 체력 극복, 연대기적 성장과정 나열.

### 6. 통과 판정 기준
- [ ] 첫 문장이 프레임을 씌우는 '주장'인가(두괄식)?
- [ ] S+T가 2~3문장 이내로 압축됐는가?
- [ ] Action이 '나(I)' 주도이며 데이터·툴·방법론으로 구체적인가?
- [ ] Result가 정량화됐거나(수치), 실패 시 학습·후속조치가 명료한가?
- [ ] '왜 경쟁사가 아니라 우리 회사인가'에 예리한 기업 분석 논리가 있는가?
- [ ] 모든 문장이 팩트 기반이라 면접 압박에서 무너지지 않는가?
- [ ] 남과 비슷한 경험에서도 본인만의 고유 관점·입체적 깨달음이 있는가(차별성)?

> **런타임 재확인:** 회사·직무 분석에 쓰는 지역 산업 클러스터 구성·채용 트렌드 등 시장 정보와 컨설팅 홍보 수치는 시의성 데이터다 — 본 방법론에 수치를 고정하지 말고 JD Browsing으로 재확인해 사용한다.


## §3 resume-en — 영문 이력서(Resume/CV) 작성 방법론

### 0. 핵심 원리 (이중 관문 + 지독한 정량화)
- 현대 이력서는 **전략 문서**다. 두 관문을 순서대로 통과해야 한다: (1) AI 기반 ATS의 파싱·키워드·시맨틱 필터 → (2) 몇 초 안에 훑는 인간 채용담당자.
- 채용 패러다임이 학위·직함 중심 → **스킬 중심(Skills-First)**으로 이동. "회사명·학위"가 아니라 "실제로 무엇을 실행할 수 있는가(Capability)"를 증명한다.
- 편향(AI·인간 모두)을 무력화하는 유일한 방어무기는 **주관 형용사 배제 + 절대적 정량지표**다. '우수한/열정적/proven track record' 같은 표현은 편향 개입 여지가 크고, '비용 $18,000 절감', '다운타임 40% 감소' 같은 수치 앞에서는 편견이 작동하지 못한다. 이력서를 **수치 데이터셋**으로 탈바꿈시키는 것이 목표.

### 1. CV vs Resume 판별
- **CV**: 학계/연구/석박사/의료 지원용 포괄 문서. 페이지 제한 없음, 논문·학회·연구비·특허·추천인 중심.
- **Resume**: 기업 취업용 압축 마케팅 문서. **1페이지**(시니어 10년+만 2페이지, UK는 '정확히 1페이지' 엄수).
- 신입/주니어/국내 외국계는 둘을 혼용. 판별 후 분량·섹션 구성을 결정한다.

### 2. 마스터 파일 전략 (Master File)
1. 모든 학력·인턴·프로젝트·대외활동·자격증·성과를 **한국어로 분량 제한 없이** 기록한 원본 DB(CV형)를 먼저 구축한다. (잊힌 성과 발굴 + 수치 복원)
2. 지원 시 해당 **JD가 요구하는 역량에 부합하는 경험만 전략적으로 선별(Strategic Selection)**해 1페이지 Resume로 압축·조립한다.
3. 효과: 작성 속도와 직무 적합성을 동시 극대화, 지원마다 재활용.

### 3. 섹션 골격 & 위계 (역순 연대기 / Reverse-Chronological)
표준 순서(경력자 기준): **헤더 → 가치제안 요약 → Work Experience → Skills → Education → (연구직만) Publications/Activities/References**
- 신입/주니어는 Education을 Experience 앞으로. 경력자는 Experience를 앞으로.
- Experience·Education 내부는 최근 → 과거 순.
- 섹션 헤딩은 **글로벌 표준 용어 엄수**: `Work Experience`/`Professional Experience`, `Education`, `Skills`. 창의적 제목('My Career Story') 금지. 경력 부족자는 'Work History' 대신 'Professional Experience'로 하고 학교 프로젝트·수료증 포함.

### 4. 성과 불릿 작성 공식 (가장 중요)
모든 불릿은 **[과거형 액션 동사] + [무엇을/어떤 방법으로] + [정량 결과]** 로 통일. 세부 골격 두 가지(호환됨):
- **구글 X-Y-Z 공식**: `Accomplished [X] as measured by [Y], by doing [Z]` — "[Z]를 실행해 [Y]로 측정되는 [X] 성과 달성".
- **동사+수치+by 골격**: `[동사(과거형)] + [정량 성과] + by + [달성 방법]`. 예: *Achieved sales growth 30% by rearranging product position & driving customer satisfaction*.
규칙:
- 주어 'I' 제거, 과거형 능동동사로 시작, 마침표 없는 불릿, 관사·군더더기 최소화.
- **모든 불릿을 숫자로 마감**: 건수·%·금액·인원·기간·팀규모·범위. before→after 대조(85%→98%)나 '%+절대금액' 병기가 가장 강력.
- 정량화가 어려운 간접부서도 예산 규모·절감 시간·영향받은 고객/직원 수 등 **합리적 범위(Ranges)·추정치**로 스케일을 제시.
- 리더십 정량화: 팀 규모, 관리 프로젝트 수, on-time/on-budget 납품, 예산 규모를 수치로. (예: *Led a team of 15 to deliver $2M in cost savings*)

### 5. 액션 동사 (Action Verbs)
- **모든 불릿을 강한 액션 동사(과거형)로 시작**하고, 약한/수동·진부 표현을 치환한다.
- 배제할 진부어(Dead Words): `Responsible for`, `Helped`, `Worked on`, `Made`, `Did`, `Fixed`, `Talked`, `Managed(단독)`, `In charge of`.
- 치환 예: Responsible for→**Managed/Led**; Helped→**Facilitated/Supported**; Worked on→**Developed/Executed**; Made·Built→**Engineered/Fabricated**; Fixed·Changed→**Revitalized/Rectified/Mitigated**; Talked→**Persuaded/Articulated/Arbitrated**.
- **동일 동사 반복 금지** — 유의어로 다양화.
- 역량군별 동사 매핑 테이블(JD 역량에 맞춰 선택):

| 역량군 | 동사 |
|---|---|
| 성과/달성 | Achieved, Attained, Exceeded, Completed, Executed, Delivered, Generated, Won |
| 리더십/관리 | Led, Directed, Orchestrated, Spearheaded, Managed, Supervised, Coordinated, Administered, Oversaw, Consolidated |
| 구축/개발/창시 | Built, Created, Designed, Developed, Established, Founded, Formulated, Initiated, Introduced, Pioneered, Launched, Engineered |
| 분석/문제해결 | Analyzed, Assessed, Audited, Diagnosed, Evaluated, Examined, Identified, Investigated, Researched, Resolved, Rectified |
| 개선/최적화/성장 | Improved, Increased, Expanded, Enhanced, Strengthened, Reduced, Simplified, Standardized, Streamlined, Optimized, Transformed, Upgraded, Overhauled |
| 커뮤니케이션/설득/협상 | Communicated, Presented, Persuaded, Influenced, Negotiated, Mediated, Facilitated, Promoted, Marketed, Advocated |
| 교육/육성 | Coached, Counseled, Instructed, Mentored, Trained, Motivated, Guided |

### 6. ATS 호환 포맷 철칙 (단순함 + 일관성)
- ATS 탈락 3대 원인을 역으로 방어: (1) 직무 필수 하드스킬/키워드 누락, (2) 최소 자격요건(경력연수·학위·라이선스) 미탐지, (3) 포맷 오류로 파서가 데이터 추출 실패.
- 파일: **텍스트 기반 PDF 또는 .docx**(지원처가 명시한 형식 최우선).
- **절대 금지**: 텍스트박스·표·이미지·아이콘·인포그래픽·다단(컬럼) 레이아웃. (표를 쓰면 ATS가 상하로 읽어 정보가 뒤섞임)
- 안전 폰트: Times New Roman / Arial / Calibri, 본문 10~12pt, 여백 0.5~1인치.
- **날짜 포맷 일관 표기**: `January 2020 – November 2023` 또는 `01/2020 – 11/2023` — 파서가 총 경력 개월수를 계산할 수 있게.
- **키워드는 시맨틱 문맥에 통합**: 흰글씨 스터핑·단순 복붙은 NLP 발달로 무효. JD 핵심 키워드를 요약란·경력 불릿의 성과 서사 안에 자연스럽게 녹인다.

### 7. 헤더 작성 & 금기
- 표기: 영문 이름(`Gil-Dong Hong`/`Gildong Hong`, 영어명은 `Gil-dong (Alex) Hong`) → 전문적 이메일(국내 도메인보다 Gmail 등 글로벌) → 국가코드 포함 연락처 → **LinkedIn URL 필수**. 개발자는 GitHub, 기획/디자인은 포트폴리오 딥링크로 'References Available' 문구를 실체 증거로 대체.
- 신입 샘플식 간이 헤더: `Gu | City, Country | Phone | Email` (파이프 구분).
- **절대 금기(차별금지 컴플라이언스)**: 사진·생년월일/나이·성별·국적·결혼여부·가족관계·주민번호 등 직무 무관 사적정보 전면 배제. 서구 차별금지법상 위반 시 스크리닝 단계에서 즉시 파기 리스크. 공인영어성적은 자리 남지 않는 한 생략(인터뷰서 검증).
- 비자 필요 시 비자 종류+만료일 명시로 스폰서십 우려 해소.

### 8. 가치제안 요약 (Value Proposition Summary)
- 구식 `Objective Statement`는 폐기. 대신 **3~4문장**으로 기업에 제공할 즉각적 효용·핵심역량을 고농축 브리핑(이력서의 '심장부', 훅).
- JD 요구역량에서 핵심 키워드 3~4개를 뽑아 본인 실무스킬·강점과 연결.
- 소프트스킬 직접 나열·corporate buzzword·다른 섹션 반복 금지, 고유 가치제안(unique value proposition)만.
- 경력 부족 주니어/학생은 연구관심사·산업문제·Can-do 태도로 경험 부족을 비전으로 상쇄. **서구권에서 겸손은 미덕 아님 — 강점을 당당히 셀링.**

### 9. 섹션별 규칙
- **Experience(핵심)**: 학위보다 경험 우위. 기관명·정확한 직함·근무지·월/연 기간을 Bold. **깊이 > 넓이** — 무관한 알바 여러 개 얕게 나열보다 지원직무 부합 핵심 경험 하나를 입체적으로 상술. 개인 기여만이 아니라 팀 리딩·부서간 협업·이해관계자 정렬(align stakeholders)·승진(progression)을 드러낸다. 전이 불가한 무관 알바는 제외.
- **Skills**: 소프트웨어명 나열 넘어서기. (가) **AI 리터러시** — 단순 'ChatGPT' 기재 무의미, 'LLM·프롬프트 엔지니어링으로 워크플로우 자동화, 처리시간 30% 단축'처럼 **AI가 만든 결과물**로 증명. (나) 소프트스킬은 명사 나열 대신 경력 불릿의 성과 서사에 녹임. 하드스킬은 `Advanced/Intermediate` 숙련도 명시, 언어는 `Korean (Native)`. 직무 핵심기술을 **카테고리로 묶고** 가능하면 스킬도 정량화. 제네릭/구식 스킬(`Microsoft Office`, `Communication`, `Teamwork`) 제외 — 최신성 부족 신호.
- **Education**: 대학명·위치·전공·학위·기간·GPA(스케일 병기, 예 `3.5/4.5`) 간략히. 관련 coursework·capstone·연구랩·지도교수·장학금으로 직무관련성 강화. 무관·오래된 항목·고등학교(어필포인트 없으면)는 제외. 팀 프로젝트는 본인 역할 개인화(`as a presenter`).
- **Certifications**: 나열만 하지 말고 실제 적용 사례와 연결. 자격증명+발급기관+취득일 기재, 만료·직무 무관 자격증 제외(재무직에 운전면허 금지). 컨퍼런스/워크샵 등 지속적 professional development로 최신성 어필.

### 10. JD ↔ 역량 매칭 (Tailoring)
- 제네릭 이력서 금지. 지원 JD를 정독해 핵심 키워드를 **Summary·Skills·Experience 전반**에 그대로 삽입. JD가 특정 용어(예: Kaizen event)를 강조하면 그 정확한 용어를 넣는다.
- **전이가능 스킬(Transferable skills)**을 해체 후 타겟 포지션 요구사항에 맞게 역으로 재조립. 마이크로 크리덴셜/산업 인증(Digital Badging)으로 스킬을 검증.
- 지원 직무별 in-demand 키워드/필수 자격증은 코칭 시점의 해당 JD·최신 공고에서 재수집한다(고정하지 말 것).

### 11. 4단계 반복 수정 프로세스
1. **한국어 브레인스토밍** — 마스터파일 뼈대 구축, 잊힌 성과 발굴·수치 복원.
2. **번역 + 벤치마킹** — '직무명 + Resume/CV Sample'로 동일 연차 현직자 프로필의 업계 전문용어·뉘앙스 차용.
3. **ATS 모의테스트 + AI 진단** — 파싱 테스트, 누락 키워드 진단(진단 툴은 런타임에 현재 이용가능한 서비스 재확인).
4. **전문가 뉘앙스 교정** — 원어민/멘토로 문화적 맥락 보정.

### 12. 국문 자소서·경력기술서 적용
동일 원리 적용: 문장을 성과 중심 동사(주도했다/구축했다/개선했다/달성했다)로 시작, '담당했다·참여했다·~를 수행함' 같은 약한 서술 지양, 각 문장에 정량 성과 결합.

### 13. 평가 체크리스트 (첨삭 시)
- [ ] 모든 성과 불릿이 강한 액션 동사(과거형)로 시작하는가.
- [ ] 약한/수동 표현('responsible for', 'helped', '담당했다')·주관 형용사·buzzword를 제거했는가.
- [ ] 동사 반복 없이 유의어로 다양화했는가.
- [ ] 각 불릿이 정량 수치(%·$·건수·인원)로 마감하고 X-Y-Z(또는 by+방법)를 충족하는가.
- [ ] 선택한 동사·키워드가 지원 JD 역량군과 일치하는가(깊이 > 넓이).
- [ ] ATS 파싱 통과: 단순/일관 포맷, 표준 섹션 헤딩, 일관된 날짜 포맷, 필수 키워드 본문 반영.
- [ ] 섹션 순서·헤더 포맷·시제가 문서 전체에서 일관적인가.
- [ ] 가치제안 요약 3~4문장이 계속 읽게 만드는 훅인가.
- [ ] 차별금지 컴플라이언스: 사진·나이·성별·국적 등 금기정보 배제.
- [ ] AI 리터러시·소프트스킬을 도구명 나열이 아니라 결과물로 증명했는가.
- [ ] 팀 활동에서 본인 역할이 개인화(as a presenter 등)되었는가.

### 14. GOOD vs BAD 대조 학습법 (첨삭 도구)
각 섹션마다 약한 예시(BAD)와 정량화·리더십 강화 예시(GOOD)를 나란히 제시해 교정한다.
- BAD: *Improved processes and increased efficiency* → GOOD: *Implemented Lean Six Sigma across operations, reducing defects by 45% and increasing productivity by 20%*.
- BAD: *Managed a team* → GOOD: *Led a team of 15, improving engagement scores by 25% and reducing turnover from 20% to 5%*.

> ⚠️ 시장/기업 통계 수치(ATS 사용률·탈락 원인 비율·스킬기반 채용률·급여/유지율 상승폭 등), 진단 툴·첨삭 서비스 이름, 직무별 in-demand 키워드/자격증, 편향 연구 결과는 **TIME-SENSITIVE** — 본문에 특정 수치를 고정하지 말고 런타임에 재검색해 인용한다(freshness_flags 참조).

## §4 interview — 면접 준비·평가 방법론

> 면접 준비/모의면접/자소서 답변 설계의 durable 코어. 기업별 배점·핵심가치·전형 시스템 등 시의성 데이터는 프리즈하지 말고 지원 시점에 JD Browsing으로 재확인(freshness 참조).

### 1. 질문 의도 역산 (모든 준비의 선행 단계)
- 표면 질문이 아니라 **평가하려는 역량**을 먼저 파악하고, 그 역량을 증명할 경험을 배치한다. 이것이 준비의 출발점.
- Durable 의도 매핑(문구는 회사마다 달라도 축은 재사용): 성장과정→직업관·Grit·가치관 / 최고목표 성취→도전·한계돌파 / 새 방식 개선→창의성·프로세스 혁신 / 실패 경험→회복탄력성(Resilience) / 원칙 준수로 손해 감수→직무윤리·컴플라이언스 / 협업 유도→소통·중재 / 강점→차별화·기여방안.
- 지원자 단어의 사전적 뜻 ≠ 면접관의 평가 정의. 예) **팀워크는 '사이좋음'이 아니라** 자발적 기여 + 긍정적 희생(배움 기회로 수용) + 타인의 협력 유도까지 포함돼야 만점.

### 2. 답변 골격: STAR-L + 두괄식 (콘텐츠×형식 하이브리드)
- **STAR-L 시간배분**: Situation·Task 짧게(약 10~15%) → **Action 최대 비중(약 50~60%)** → Result(약 20~25%, **반드시 정량 지표**) → Learning(교훈·이후 적용). Action은 '우리'가 아닌 **'내가' 한 구체 행동**. Result에 수치 없으면 치명적 감점.
- **두괄식 넘버링**: 결론 먼저("네, 저는 ~입니다") → 근거 넘버링(첫째/둘째) → (마무리). **답변 3~4문장이 적정**, 길어지면 역효과.
- 하이브리드 원칙: **필살기(내용)로 찌르고 두괄식(형식)으로 막는다.** 콘텐츠 없는 구조는 공허, 구조 없는 콘텐츠는 무질서.
- 획일화 경계: 전부 두괄식이면 '학원 기계식' 인상 → 두괄식·미괄식·에피소드형을 상황에 맞게 혼합.
- 서술 규칙: 수동태·'배웠다' 지양 → **'기여했다/성과를 냈다/통찰을 얻었다'** 등 능동·성과 동사 우선, 수치를 문장 앞에 배치.

### 3. 필살기(Killer Move) 제련 — 경험 자산화
- **필살기 3요소**: ① 유사경험(지원 직무와 유사한 현장 — 길이·스펙 아닌 **질·밀도**, 단 하루 알바도 유효) + ② 성공경험(**수치화된 성과**, '열심히 했다' 금지) + ③ 인사이트(경험→미래 성과로 잇는 사업적 통찰).
- **경험 자산화 파이프라인**: 관찰(동선·패턴·불만) → 기록(날짜·수치·특이사항) → 재해석('왜 발생했나' 가설·검증) → 인사이트. 화려한 이력 없어도 일상 경험을 치열히 관찰·기록하면 고스펙보다 구체적 역량 어필 가능.
- **경험 선별**: Project/Role/Result로 경험 리스트를 만들고, 면접관 니즈와 연결되는 **임팩트 큰 3~5개만** 필살기로 확정. 니즈 무관 경험은 많아도 무용.

### 4. 대비 맵: 면접 3단계 × 인성 5카테고리
- **단계별 축**(톤·깊이를 달리함): 1차 실무/기술 = 전공지식 정확성 + 실무 문제해결력 / 2차 임원·인성 = **태도·가치관·조직적합성(Culture Fit)** / 영어·글로벌 = 소통력 + 비전공자에게 기술 설명 + 직무 전문성.
- **인성면접 5카테고리 — 축별 대표 경험 1개씩 준비**: ① 협업/갈등관리 ② 가치관/태도(워라밸·정직vs성과·롤모델) ③ 직무적합성/로열티(경쟁사 대비 강약점·진짜 이직사유) ④ 상황대처(데드라인 충돌·부당 지시·번아웃) ⑤ 창의성/도전(관행 타파·페르미 추정·AI 직무대체 대응).

### 5. Core 빈출 문항 답변 공식 (체크리스트)
- **1분 자기소개 = 엘리베이터 테스트(면접 전체의 목차)**: 장황한 성장과정 금지. **한 단어/컨셉 정의 + 핵심역량 2 + 필살기 선언 + 입사 후 포부** → 후속 질문을 내 필살기 영역으로 유도.
- **Why Us**: '경쟁사가 아닌 우리 회사여야만 하는 구체적 이유'(제품/서비스 이해, 해결하고 싶은 Pain Point, 가치관 부합).
- **성격 장단점 = 메타인지 증명**: 결점 색출이 아니라 '객관 인지 후 개선하는가'. 장점50%+단점50%, 치명적 단점 회피, **솔직 인정 → 객관 분석 도구 활용 → 개선 행동 → 개선 결과**.
- **팀워크**: 자발적 기여 + 긍정적 희생 + 협력 유도 3요소 모두 포함된 에피소드.
- **실패 경험**: 실패 + 배운 점 + **이후 삶/업무에 어떻게 적용했는지까지 완결**(Learning).
- **역질문/마지막 할 말** 반드시 준비.
- **주도성의 역설(과유불급)**: 안전·규정 중시 직무에선 과한 주도성이 리스크로 감점 → '우직한 책임 완수'로 **직무 특성에 맞춰 재해석**. 즉 모든 역량 강조는 직무 적합성 필터를 통과해야 함.

### 6. 평가 루브릭 해독 — 지원사 프레임에 정밀 매핑
- **원리(durable)**: 엘리트 채용은 직관 면접에서 벗어나 **구조화된 정량 루브릭 + 편향 제거 + 의사결정 권력 분산**으로 이동. 지원자는 기술 스택 나열이 아니라 **지원 회사 고유의 평가 매트릭스·철학을 먼저 해독한 뒤 자기 경험을 그 프레임에 매핑**해야 통과.
- **글로벌 빅테크형 채점 패턴**: 축별 정수 척도 절대평가(예: 알고리즘/코딩/커뮤니케이션/문제해결), Hire/No-Hire 단계 척도, 만난 적 없는 위원회/제3부서 심사관의 종합·거부권. → **가중치·배점·원칙 개수는 시의성**이므로 지원 시점 재확인.
  - 코딩 면접 durable 채점축: 최적해·트레이드오프 설명, 문법오류 0·가독성·프로덕션 수준, **사고과정 실시간 투명 설명(침묵 코딩=감점)**, 엣지케이스 논의 여유.
  - Depth over breadth: 범용 알고리즘보다 도메인 극한 전문성 + 시스템 제약 하 **우아한 실패(graceful degradation)** 설명. 표면 답변=탈락 신호.
  - 성장 마인드셋(Learn-it-all): 과거 실패 수용, 비판적 피드백을 성장 도구로, 공감·포용. 오만·비협조 태도는 이전 점수 무효화.
- **국내 대기업형(하이브리드)**: 직무 역량 + **전인적 인성·조직 조화 + 경영철학 공명**을 종합 수치화. 자소서·답변이 지원 팀의 실제 사업 목표 및 회사 핵심가치와 철학적으로 일치하는지 서술. 정신자세·예의·품행이 임원 면접 당락 변수. → **각 사 5대 품성·경영철학·PT 루브릭 문구/배점은 시의성**, 지원 시점 수집.
- **자소서/서류 durable 규칙**: '팀을 관리함'(저평가) → '[행동/기술스택] + [정량 결과 %] + [비즈니스 임팩트]'(고평가). 최근 2~3년 성과 상단 배치(Recency 가중). AI 최적화하되 **실제 역량 반영선까지만**(외부 데이터 교차검증에서 허위 적발 시 배제 → Truth Tier: 과장 금지·검증가능성 우선).

### 7. 압박·후속질문 방어 규칙
- 핵심 경험은 **3~4번 파고드는 후속 질문(왜 그렇게 결정했나)까지 방어** 가능하게 준비.
- 압박에 굴복해 핵심가치를 포기하거나 이전 답을 번복하면 **'확신 부족'으로 탈락**. 유연성은 보이되 핵심 철학은 **논리로 방어**.
- 지적·반박 질문엔 즉각 반격 금지 → "그 부분도 충분히 일리가 있습니다"로 **수용·경청 후 내 논리와 연결**(내가 틀릴 수 있음 인정 = 소통역량 핵심).
- 실패 서사는 회피하지 말고 **명확히 인정 + 교훈(vocally self-critical)** = 성장 증거로 프레이밍.

### 8. 모의면접 채점 기준 (durable 루브릭)
- 답변 구조: STAR-L 준수 / Action 비중 / **Result 정량 지표 유무** / 두괄식·3~4문장 간결성.
- 콘텐츠: 필살기 3요소(유사·수치화 성공·인사이트) 충족, 단순 실행가(Doer)를 넘어 통찰(Thinker) 증명.
- 적합성: 질문 의도에 맞는 역량 배치, 직무 특성에 맞춘 재해석, 회사 프레임(가치·철학) 매핑.
- 태도: 압박 하 핵심가치 방어 + 경청·수용 + 실패 인정.
- 냉정 진단 원칙: 근거·수치 없는 주장, 팀 성과 개인화, 과장은 감점. 애매하면 낮게.
- 준비 프로세스: 빈출 문항을 나침반 삼아 질문별 STAR-L 스크립트를 사전 작성 → 무한 모의면접으로 실전화. (직무 성과와 무상관인 수수께끼형 브레인티저는 폐기된 관행 — 실증적 정량 증명만 유효.)


## §5 market-trend — 채용 패러다임 & 컨설턴트 통합 방법론

세 컨설턴트 방법론(성과 채굴·구조 포장·총보상 협상)과 채용 트렌드 분석에서 추출한 **재사용 가능한 코칭 원리**. 시장 수치·기업명·연도 라벨은 본문에 고정하지 말고 런타임 JD Browsing으로 갱신(→ freshness).

### 0) 관통 프레임 (모든 태스크의 상위 렌즈)
- **평가축 전환**: `Can do(할 줄 아는가) → Will do(어떤 마음으로) → How to adapt(어떻게 적응)`. 채용은 점수순 **필터링이 아니라 조직 맥락↔지원자 가치의 매칭/핏**. 서류·면접은 스펙 나열이 아니라 **기업 문제↔내 기여를 잇는 제안서/내러티브**.
- **자기 정의**: 수동적 '지원자'가 아니라 **노동력을 비즈니스 자산으로 파는 문제해결 파트너**. "어느 대학"보다 "무엇을 할 줄 아는가"를 증명 — 직무경험 자체가 최고 스펙.
- **통합 스파인 3단**: ①채굴(콘텐츠=필살기 발굴) → ②포장(형식=평가자 언어로 구조화) → ③매각(총보상 협상으로 가치 실현). 태스크는 이 순서의 어느 지점에 위치하는지로 배치.

### 1) 경험 채굴 — 필살기 진단
**공식**: `필살기 = 유사경험 ∩ 성공경험 ∩ 인사이트` (세 축 교집합; 결여 축을 찾아 보완 유도).
- **유사경험**: 지원 직무의 Key Task와 맥락 **밀도**가 핵심. '동아리 회장'=리더십일 뿐 마케팅 유사경험 아님 → 실제로 팔아본 경험이어야. (편의점 알바라도 재고 시스템 개선이면 SCM 유사경험으로 성립.)
- **성공경험**: '열심히 해서 좋은 결과'는 **기각**. 반드시 정량 지표(%, %p, 배수, 금액)로 수치화. 성공경험은 조직이 바뀌어도 전이된다고 전제.
- **인사이트**: '운/팀원 덕분'은 인사이트 아님(기각). 성공 원인을 관통하는 **재현 가능한 자기 법칙**을 설명해야 — 지원자가 Doer가 아닌 Thinker임을 증명.

**경험 3분류**: 필살기(직무 직결) / **빌살기**(직무 무관하나 태도·기초역량 어필용, 잠시 빌려 씀) / **밉살기**(전문성 훼손·잦은 이직 등 → 방어논리 구축 or 삭제).

**마인드셋 교정**: 발화에서 '학생 마인드'(배우겠습니다·성장하겠습니다·기회 주시면 → 기업엔 교육비=비용) 탐지 시 '비즈니스 마인드'(이런 성과를 냈다·수치로 증명한다·문제 해결에 기여 → 즉각 ROI)로 교정. 경력직은 '배우겠습니다'가 아니라 **'기여하겠습니다'**.

### 2) 시장 진단 포지셔닝 — 3C → 4P
- **3C(문제진단, Why Me)를 4P보다 먼저**: Customer=지원 기업을 '채용해주는 곳'이 아니라 **Pain Point를 가진 고객**으로 정밀 타격 / Competitor=타 지원자·AI 대비 차별점(현장성·복합 문제해결·인간적 연결) / Company=나의 필살기.
- **4P(해결책, How I Help)**: Product(역량)·Price(가성비·기대수익)·Place(기여 채널)·Promotion(어필).
- **검증**: 자기주장('저는 꼼꼼합니다') **전에 3C 진단이 선행**했는가, 3C 문제와 4P 해결책의 **논리연결이 타당**한가. Pain Point는 현직자 인터뷰·기업 신년사·ESG 보고서로 발굴하고 그에 맞춘 구체적 역질문을 준비.

### 3) 성과 정량화 — "설명 가능한 성과"
- 주관 형용사 배제. **포트폴리오화 공식**: `어떤 툴을 써서 → 어떤 문제를 해결해 → 얼마의 효율을 냈는지` + 나의 **역할·기여도(%)** 명시.
- **지표 어휘**: 효율(시간 단축률·비용 절감·공수 절감) / 품질(오류검출률·수율·데이터 정확도·재작업률↓) / 임팩트(전환율·체류시간·이탈률·매출 기여도) / 직무특화 지표.
- **포트폴리오 5단계**: `Context(개요) → Role(역할·기여%) → Action(행동) → Result(수치) → Insight(배운 점)`. 이력서=재료명세서, 포트폴리오=완성 요리. 전 직무 공통 적용.

### 4) 역량 언어 변환 — 평가자 taxonomy
산발적 장점을 **평가자가 이해하는 카테고리 언어**로 번역.
- **핵심역량 5분류**: ①대인관계·소통(갈등관리·설득·팀워크) ②업무수행능력(전략적 사고·정보수집·분석·기획) ③조직적합성·태도(성실·책임·**주도성**=시키지 않은 일을 찾아 수행) ④윤리의식 ⑤적응력(혁신·긍정·고객지향).
- **전통역량 → 미래역량 매핑**: 툴 사용 → AI 협업지능 / 정답 찾기 → 문제 정의·구조화 / 대면 보고 → 맥락적 연결 / OJT → 학습 민첩성 / 성실·야근 → 데이터 기반 성과 입증. JD의 요구를 이 미래 축으로 치환해 소구점 설정.
- **'일잘러' 3대 역량**(자기진단 축): AI 협업지능(질문 설계+AI 결과의 오류·편향 비판적 검증+인간 판단 보완) / 설명 가능한 성과(의사결정 근거·변수 대응을 육하원칙으로 복기) / 맥락적 연결자(비동기 커뮤니케이션 명확성+갈등 조정).

### 5) 답변 설계 알고리즘 (면접·자소서 공통)
- **두괄식**: 첫 문장에 결론(핵심역량 선언) → 면접관 머릿속에 인덱스를 먼저 심음. 예) "제 강점은 진상 고객마저 단골로 만드는 설득력입니다. 편의점 근무 당시…"
- **넘버링**: "이유는 첫째~, 둘째~"(근거 2~3개) → 구조화 인상 + 발언권 확보. 답변 길이 **40~50초**. 마무리 "따라서 저는 ~할 수 있습니다."
- **진화된 STAR**: `S-T-A-R + Insight(배운 점) + Reflection(아쉬웠던 점)`. 실패 경험을 솔직히 회고하고 배운 점을 명확히 하면 가산점.
- **내용 AND 태도**: 합격 = 내용(Content) ∩ 비언어(아이컨택·톤·자세·자신감). 밸런스 깨지면 합격률 급락 — 텍스트 코칭 시 "내용 좋아도 시선 불안하면 신뢰 상실, 거울 보고 연습" 메타조언.
- **팀워크 재정의**: '사이좋게'가 아니라 '공동목표 위한 자발적 희생+건설적 갈등'. 구조 = 위기 → 역할 외 업무 자처(희생) → 협력 유도(기여) → 목표 달성(수치).
- **리스크/약점 대응**: 변명·외부귀인 금지 → 약점 솔직 인정 → 전환점(Turning Point) → 현재의 보완 노력(자격증·경험)으로 신뢰 형성.

### 6) 서류 통과 (ATS/AI) 3원칙
ATS는 키워드+문맥의 **의미론적 유사성**을 평가. ①**정량화**(모든 성과 숫자화) ②**도구 명시**(AI툴·협업툴 구체 기재 → 디지털 리터러시 증명) ③**과정 중심**(Process over Product, 문제 정의·해결의 사고과정 노출).

### 7) 면접 Green / Red Flag 체크리스트
- **Red**: 팀 성과를 '제가 다 했습니다'로 독차지 / 실패·퇴사를 상사·회사·팀원 탓(외부귀인) / 역질문 없음 or 연봉·복지만 질문 / 구체 Action·Result 없는 모호한 일반화 / 화상면접 소음·시선 회피·지각 등 에티켓 결여.
- **Green**: 진화된 STAR+회고 / 회사 Pain Point를 짚고 기여방안을 제시하는 **컨설턴트형** 답변 / 모르는 질문에 아는 척 대신 **지적 겸손**(인정+힌트 요청) / 성과를 ROI·전환율 등 수치로 증명 / 깊이 있는 역질문(지원 진정성 신호).
- 문화는 순응(Fit)을 넘어 **Culture Add**(고유 관점으로 다양성 기여) 지향.

### 8) 세그먼트 튜닝 (durable 축; 개별 기업 키워드는 런타임 갱신)
- **연차별 기대치**: 신입=잠재력보다 기초 직무 근육(인턴·프로젝트)·완결성·그릿, '배우러 왔다' 프레이밍 금지(예비 프로로 포지셔닝) / **중고신입(3년 미만)**=즉시 투입 가능성(교육 없이 OJT만으로 실무) / 경력직=구체적 프로젝트 성공+수치 성과.
- **연차별 역량 모델**: 3년=즉시전력(Delivery·Basic Skill·Communication) / 6년=작은 리더십(Problem Solving·Mentoring·Cross-functional) / 9년=전략(Strategic Thinking·Team Building·Risk Mgmt) / 12년+=임팩트(Visioning·Crisis Mgmt·C-Level Communication). 시니어는 오너십을 회색지대까지 확장하고 **비즈니스 언어(매출·비용·리스크)**로 소통.
- **기업유형 아키타입**: 대기업=시스템 내 프로세스 개선·수치 증명+준법 / 스타트업=자기주도·급진적 솔직함·미션 진정성 / 외국계=리더십 원칙·컬처메모 내재화한 상황질문 대비 / 강소·중견=다기능 멀티플레이어+장기 근속 의지.
- **전형 단계별 핏**: 서류·1차 실무 = Job Fit이 당락 / 최종 임원 = Culture Fit이 핵심 → 단계별 강조점 전환.
- **지원자 코칭 강조점**(페르소나 아님, 배합 힌트): 저스펙/신입=필살기 채굴 비중↑+두괄식으로 형식 커버 / 고성과 경력=성과 중심 경력기술서+사전 협상 포지셔닝 / 긴장형=넘버링·두괄식 기계적 암기로 구조 확보(내용은 심플).

### 9) 커리어 설계 로드맵
- **[전공 계열 – 추천 직무 – 타깃 산업 – 핵심 준비사항] 4열 매칭**: 막연한 전공 기반이 아니라 산업을 타게팅한 뒤 역산으로 액션아이템 채움.
- **문과 피보팅 3종**: ①데이터 리터러시(Python/SQL/Tableau 중 1+) ②도메인 타게팅('아무데나' 금지, '핀테크 규제 전문가'처럼 특정) ③프로젝트 포트폴리오(부트캠프·인턴; 서류 통과율을 좌우).
- **이공계 심화**: 하이엔드 직무는 석·박이 사실상 지원 자격 / GitHub·기술 블로그로 코드·연구 기록 / 산학 인턴·랩실은 해당 기업 직행 지름길.
- **커리어 사다리 'Start Small, Aim High'**: 대기업만 고집해 공백기 늘리지 말고 직무 연관 강소·스타트업에서 1~2년 경력 후 대기업 주니어로 점프(중고신입 루트). "완벽한 곳보다 어디서든 시작하는 것이 중요."
- **직무 프레임**: Builders(엔지니어·R&D) vs Scalers(비엔지니어·기술 사업화) — **전 직군 데이터 리터러시 공통 요구**.

### 10) 연봉협상 — 총보상 기반 논리 거래
- **총보상 공식**: `Total Comp = 기본급 + 성과급(PS/PI/OPI) + 현금성 복지(식대·포인트·통신비) + 주식(RSU)`. 통장 월급이 아닌 '영끌' 총액을 협상 기준점으로. 기업 제시액과 1:1 비교 시 **누락 항목 체크**, 인상률은 기본급 기준.
- **5단계 알고리즘**: ①준비(영끌 계산+마지노선 **BATNA**) → ②탐색(연봉 질문 시 **역질문**으로 예산 파악) → ③제안(예산 파악 성공→상한 근처 레인지 / 실패→목표 레인지) → ④조율(1차 오퍼 수신) → ⑤판단(만족→수락 / 큰 차이→정중 거절+성과데이터 재전송 / 미세 차이→핑퐁 자제 후 수락).
- **3 스크립트**:
  - 선제방어 역질문(Winner's Curse 방지): *"제 희망액도 중요하지만, 귀사의 내부 규정과 해당 포지션 예산 범위(Budget Range)가 궁금합니다. 알려주시면 긍정적으로 검토하겠습니다."*
  - 레인지 전략(단일 숫자 대신 범위 → 유연성+압박 완화): *"현재 총보상·시장 가치·이직 리스크를 고려해 OO~OO 사이를 희망합니다."* 첫 결재(품의)에 반영되도록 **초기에 명확히** 전달.
  - 품의 시스템 이해: 협상 상대는 담당자 개인이 아니라 **결재 시스템**. 첫 제안에 명확한 근거(원천징수영수증·성과 포트폴리오·경쟁사 오퍼)+희망 범위를 확실히 실어 첫 품의에 반영. 목표치와 미세 차이(핑퐁 스트레스·채용 취소 위험)면 커리어 성장 가치 보고 수락.
- **이직 사유 재구성**: 회피성('연봉 불만') → 미래지향('직무 확장'·'산업 성장성').

### 11) 실행 루틴 (durable)
- **GDR**: Goal(막연한 '취업' 아닌 '오늘 자소서 1개/현직자 인터뷰 1회' 구체 목표) + Routine(반복 일상) + N(지원 횟수 지표). 성과는 양적 투입(Input)이 전제.
- **경신스(경제신문 스크랩)**: 산업 트렌드·업계 용어로 면접관과 대화 가능한 비즈니스 문해력 배양 + 지원 기업 최신 이슈를 답변에 녹임.
- **면접 복기록**: 받은 질문 / 나의 답변 / 면접관 반응 / 놓친 필살기 → 피드백 루프로 다음 면접에서 밉살기 축소·필살기 강화.

### 통합 평가 체크리스트 (2단 진단에 투입)
1. 필살기 3요소(유사·성공·인사이트) 모두 충족? 2. 성공경험 정량 수치 존재? 3. 인사이트가 재현 가능한 원리? 4. 자소서에서 3C(문제) 선행 후 4P(해결) 논리연결 타당? 5. 산발 장점이 5대/미래역량 언어로 변환? 6. 답변 두괄식+넘버링+40~50초? 7. 내용 AND 태도 둘 다? 8. Red Flag(공 독차지·외부귀인·역질문無) 회피 & Green Flag 확보? 9. 총보상 누락 항목 점검?

### freshness — 런타임 JD Browsing으로 재수집(수치 동결 금지)
아래는 방법론이 아니라 **시의성 데이터**다. 본 모듈에 고정하지 말고 지원 시점에 최신 공고·컬처메모·신년사·시장자료로 갱신할 것:
- 산업별·기업별 연봉/총보상 테이블, 성과급 비율(PS/PI/OPI), 주식보상, 개발자 등 직무 격차·이직 인상률 통상치
- 8대 핵심 산업·클러스터·유망 기업 리스트, 대학/학과별 취업률, 중고신입 비중·수시채용 활용률·신입/경력 채용 비율
- 개별 기업 인재상 키워드·신년사·컬처메모·리더십 원칙 문구, 채용 프로세스명(적성검사)·AI 면접 솔루션 브랜드
- 정부 선정·제도(히든챔피언·메가클러스터·규제법 등) 및 시장 진단 라벨('대분열'·'초양극화' 등)
- 지원 기업·산업의 최신 이슈(경신스용)와 지원자 개인의 실제 시장 연봉 벤치마크(협상 근거)

## §6 roadmap — 커리어 로드맵·자가진단·직무전환·포지셔닝

### 0. 대전제(모든 로드맵의 프레이밍)
- **리퀴드 노동시장**: 직무(Job)가 아니라 스킬·태스크 중심. 직무를 "고정 역할"이 아니라 "유동적 태스크 집합"으로 본다.
- **3층 희소성 공식**: ① AI 활용력 = 기본기(Default, 깔고 시작) → ② 그 위에 AI 대체불가 역량(공감·윤리판단·복합소통) → ③ 이질 전공·스킬 융합. 기술 평준화와 인간 차별화 두 극단을 동시에 충족.
- **자기정의 리프레이밍**: 부서·직급("마케팅팀 김 대리")이 아니라 "해결 가능한 문제 + 필살기 스킬"("데이터 분석 가능한 퍼포먼스 마케터")로 정의. 필요 시 포지셔닝 직함으로 승격(그래픽 디자이너→비주얼 솔루션 디자이너 등).
- **Job Crafting 질문 전환**: "이 직무가 AI로 대체되나?"가 아니라 "어떤 태스크가 자동화되고, 나는 어떤 고부가 태스크로 이동하나?". 태스크 3분류로 재설계 — 반복·규칙(AI 위임) / 예외처리·전략수립(인간 핵심) / 관계·협업(인간 고유).

### 1. 거시환경 스캔(로드맵 착수 전 '판 읽기')
- **3대 메가트렌드 스캔**: 빅 블러(산업경계 소멸·융합) · 인구구조 변화(산업별 인력 수급 불균형) · 직무 반감기 단축(기술 유효기간 급감).
- **규칙**: 목표 "산업"이 아니라 "융합되는 방향의 역량"을 겨냥. (자동차가 목표여도 기계공학보다 SW·클라우드 역량이 값짐.)
- **New Skills Triad**를 지속 학습 목표로: AI 문해력 · 가상 지능(원격·디지털 협업) · 탄소 지능(지속가능성). → 구체 시장규모·수급 수치·기업 사례는 런타임 재수집.

### 2. 자가진단(self-diagnosis) — CORE 4역량 + 예/아니오 체크리스트
CORE: **Technical Monopoly**(AI 모방불가 High-Touch·현장 암묵지) · **Connective Intelligence**(이질 두 분야 연결) · **Personal Branding**(회사 명함 아닌 '이름 석 자'가 브랜드) · **Agility**(프로젝트 단위 기그 적응).
- AI 리터러시: 업무 시간의 유의미한 비율을 AI로 단축하고 있는가?
- 미래 전망: 내 산업은 향후 성장/소멸? — 감이 아니라 데이터 기반 판단인가?
- 전문성: 회사 계급장을 떼도 나를 찾는 외부 클라이언트가 있는가?
- 연결성: 서로 다른 두 분야 이상을 융합해 하나로 설명할 수 있는가?
- 브랜딩: 온라인에 전문성을 증명할 축적된 기록(포폴·글·영상)이 있는가?

### 3. 융합·희소성 설계(Base+Plus / Slash Effect)
- **공식**: 주 전공·직무(Base·본캐) × 이질 스킬(Plus·부캐) = 타겟 직무 창출. 두 분야를 하나로 설명할 수 있으면 대체불가성↑.
- **산출물**: 결합 공식표 = 융합유형 | Base+Plus | 타겟 직무 | 전략 포인트.
- **패턴 예시(evergreen)**: 바이오+통계=생물정보학, 디자인+프론트엔드=UX엔지니어, 인문+AI=AI서비스기획/테크니컬라이터, 경영+산업공학=기술영업/SCM, 간호+코딩=헬스케어앱기획, 회계+ESG=지속가능경영컨설턴트.
- **실행 경로**: 학생=학점 손해 감수 부·복수전공 + 타 전공 팀 프로젝트 / 현직자=퇴근 후 이종 사이드 프로젝트, 사내 Job Posting으로 유관 부서 이동.

### 4. 연차별 성장 사다리
- **주니어**: 압도적 생산성·즉시전력 증명(AI로 작업시간 대폭 단축, 반복업무 자동화 실적).
- **미드**: 비즈니스/PM 관점 — ROI·숫자로 경영진 설득.
- **시니어**: 거버넌스·아키텍처·리더십으로 기술을 비즈니스 가치로 연결(C-level 시야).
- **경력직 골든타임 인지**: 저연차 수요 최다, 중간 구간 기술 완성·연봉 도약, 고연차는 팀리딩 vs 전문가(Expert) 트랙 분기. (연차 경계·연봉 수치는 런타임 재수집.)

### 4.5 다중 경로 추천 엔진 (Path Recommender) — 하나가 아니라 여러 경로 제시
로드맵의 핵심 실패는 **경로를 하나로 좁혀 제시**하는 것이다. 사용자의 커리어 히스토리(뱅크)를 입력으로 **가능한 여러 발전 경로를 생성·점수화·순위화**해 **비교 가능한 선택지**로 제시하고, 사용자가 고르면 그 경로만 §5 상세 로드맵으로 전개한다. "당신의 길은 이것"이 아니라 **"당신이 갈 수 있는 길들은 이렇고, 각각은 이런 준비가 필요하다"**.

**(1) 경로 아키타입 (히스토리에서 생성 — 최대한 넓게)** — 뱅크의 도메인·필살기·스킬·성과를 씨앗으로 아래 축을 **모두** 탐색해 후보를 만든다(해당 없는 축은 제외하되 임의로 좁히지 않는다):
- **A. 도메인 심화(Specialist/Expert)** — 현 직무를 더 깊게, IC 최고 전문가(Technical Monopoly). 
- **B. 매니지먼트(Leadership)** — 사람·조직·전략(팀리딩→임원). §4 시니어 트랙.
- **C. 인접 직무 확장(Expansion, not Pivot)** — 도메인 유지 + 인접 직무(§7 1역량→N직무 매핑). 기존 강점 위에 얹기.
- **D. 디지털·데이터 피벗** — 현 도메인 + 데이터/AI/SW 무장(하이브리드 엔지니어 삼각형). "양손잡이".
- **E. 산업 환승(Reskilling)** — §7 스킬 1:1 번역으로 성장 산업 이동(정체→성장). 갭 큼·고위험·고성장.
- **F. 융합·희소성(Base+Plus)** — §3 슬래시 결합으로 신규 포지션 창출(대체불가).
- **G. 독립·슈퍼개인** — 자문·강연·콘텐츠·창업(§2 CORE Personal Branding·Agility).

**(2) 경로별 4축 점수화(0–100, 주관 금지)** — 각 후보 경로를 아래 4축으로 채점 후 가중 합산. 발굴 보드(jd-discovery)의 적합도 철학을 커리어 성장에 이식:
| 축 | 무엇을 보나 | 가중 | 산정(객관) |
|---|---|---|---|
| **근거 적합** | 뱅크 경험이 이 경로를 얼마나 뒷받침하나 | ×0.35 | 관련 필살기/성과 원자 수·[T1] 비율 |
| **갭 실현성** | 도달까지 스킬 갭이 작아 실현 쉬움(역: 갭 클수록 낮음) | ×0.25 | As-Is/To-Be Hard·Soft 갭 항목 수 |
| **시장 성장성** | 목표 직무·산업의 수요 추세(JD Browsing) | ×0.25 | 성장/정체/소멸 + 채용 시그널(Tier·조회일) |
| **지향 부합** | 사용자가 밝힌 방향·가치와 맞나 | ×0.15 | 사용자 명시 선호(없으면 중립 50·`[확인 필요]`) |
- **가중 합 = 경로 적합도.** 등급 80+ 유력 / 65+ 유망 / 50+ 검토 / <50 도전(고갭·고위험). **+ 확신도.** 각 축 근거 없으면 Score Caps(evaluation §2)·"지향 부합"은 사용자 미확인 시 `[확인 필요]`.
- **점수는 배제가 아니라 정렬용** — 낮은 점수 경로도 **목록에 남긴다**(고위험·고성장은 점수는 낮아도 사용자가 택할 수 있음). 시스템이 가능성을 닫지 않는다(jd-browsing §0 자유도 보존과 동일 철학).

**(3) 경로별 연차별 목표(preparation → milestone)** — 각 후보 경로마다 §5 Phase 골격 + §4 연차 사다리로 **연차별 [목표·실행·측정지표]**를 붙인다(신입 0-1-3 / 시니어 3-5-10, §8 0-1-3 골격 재사용). 이게 "추천으로 끝나지 않고 준비 방법까지 잇는다"의 실체. 각 마일스톤은 증명 가능(자격·산출물·수치).

**(4) 산출 = 다중 경로 비교 보드** — 여러 경로를 **한 화면에서 비교**(경로 유형·목표 직함·적합도 점수·4축 바·한줄 이유·연차 타임라인·리스크). 사용자가 **선택** → 그 경로만 §5 상세 로드맵으로 전개. 미선택 경로는 뱅크에 '검토 후보'로 보존(나중에 재검토). 비교는 사용자 판단을 돕는 것이지 대신 결정하지 않는다.

### 4.6 직급 로드맵 (Rank Ladder) — 현 조직 내 승진 트랙 (일반 베이스 + 회사별 인터뷰 맞춤)
§4.5가 "**어느 직무·방향**으로 갈까"(직무 로드맵)라면, 직급 로드맵은 "**현 조직(또는 목표사) 안에서 어떤 직급 단계를 어떻게 밟나**"다. 두 축은 직교(直交)한다 — **방향(직무) × 높이(직급)**. 사용자는 둘 다, 또는 하나만 원할 수 있으니 어느 축인지 먼저 확인.

**(1) 일반 베이스 아키타입 (evergreen 뼈대 — 명칭·요건은 회사마다 다름)**
- **국내 대기업 전통형:** 사원 → 대리 → 과장 → 차장 → 부장 → 임원(상무·전무·부사장·사장).
- **직급 단순·수평형(최근 다수):** 사원 → 선임 → 책임 → 수석 (또는 매니저/프로 단일 호칭·역할 등급).
- **테크/개발:** 주니어 → 미들 → 시니어 → 스태프/프린시펄(IC) ↔ 팀장/EM(관리) — **이원 트랙**.
- **외국계/글로벌:** IC 라더 ↔ Manager 라더 이원화(레벨별 스코프·기대가 문서화된 경우 많음).
- **연구직:** 연구원 → 선임 → 책임 → 수석. **공공/공기업:** 급수 체계.
> 베이스는 **뼈대일 뿐** — 실제 명칭·단계 수·경계·요건은 (2) 인터뷰로 확정한다(고정 금지).

**(2) 회사별 인터뷰로 맞춤 (핵심 — 일반론을 실제 조직에 정합)**
회사 구조가 다르므로 **물어서 만들어 간다**. 최소 질문 세트:
- 직급 체계: 실제 직급 명칭·순서·**현재 직급**.
- 단계별 승진 요건: 최소 연차(체류연한)·인사고과 누적 등급·승진 심사/시험·다면평가·필수 자격/어학·정량 성과 커트라인.
- 승진 방식: 자동 호봉 / 심사 / TO 경쟁 / 발탁. 평가 주기·평가자.
- **다음 목표 직급 + 목표 시점.**
→ 답을 **직급 라더로 구조화** + 회사 컨텍스트를 뱅크(`.private/`, 회사 내부 구조 = 개인 컨텍스트)에 저장. 모르는 요건은 **`[확인 필요]`**로 두고 사용자가 사내 규정·인사팀으로 확인 후 [T1] 승격.

**(2′) 공개 자료 보강 (JD Browsing — 선택, 인터뷰 전/중)**: 공개 웹으로 베이스를 회사 근거로 상향한다. **직급 체계·개편**(뉴스·공식 = S/A), **상위 조직/사업부**(DART·IR·보도 = A), **공식 현직자 인터뷰**(A, 정성)는 유효 — 일반 베이스보다 정확. 단 **구체 승진 요건**은 공개 신뢰도가 낮다(블라인드·카페 후기 = B/C·게이트·편향·구식·사업부별 상이) → **후보 시그널로만** 수집해 (2) 인터뷰에서 확인. **유출 내부문서·팀 상세 조직도 미사용.** 확정 근거는 **사용자 진술·사내 규정(T1)** — 공개 자료는 인터뷰를 날카롭게 하는 입력이지 확정 아님. (jd-browsing §3·§5)

**(3) 산출 = 직급 라더 (단계별 요건 → 연차별 준비)**
각 직급 단계에 붙인다: **승진 요건**(연차·고과·심사) + **연차별 준비**(목표·실행·측정지표, §5 골격 재사용) + **평가 대비**(고과·다면평가에서 무엇을 증명). **현재 직급 → 다음 목표**까지 우선 전개, 이후 단계는 요건만.

**(4) 두 축 합성**
직무 경로(§4.5) 선택이 직급 트랙을 규정한다 — 예: '전문가(IC)' 경로 → IC 라더, '매니지먼트' 경로 → 관리 라더. 두 축을 함께 제시해 사용자가 **방향과 높이를 같이 설계**하게 한다.

**Truth Tier·근거:** 일반 베이스 = 예시(동결 금지). 회사 요건 = 사용자 진술(문서화=T1, 구두=T2), 미검증=`[확인 필요]`. 승진 연차·TO·고과 커트라인 등 회사별 수치는 프리즈 금지(사용자·사내 규정에서 재확인).

### 5. N년 단계별 로드맵 설계 — 각 단계 [목표·실행·측정지표] 3요소 필수
**Phase 골격은 선택 경로 아키타입별 3종(§4.5 경로 선택이 규정 — 임의 주입 금지):**
- **매니지먼트 / 직급 라더(§4.6):** 승진 요건 충족 → 담당 스코프·예산·인원 확장 → 리더십·조직 성과 증거 축적.
- **IC 심화(전문가):** 기술 독점성(High-Touch·희소 역량) → 아키텍처·표준 주도 → 대외 레퍼런스(발표·기고·오픈소스).
- **독립·슈퍼개인:** 아래 3단계(디지털무장 → 하이브리드·브랜딩 → 포트폴리오화·수익 다각화). **이 부업 수익화 서사는 독립 경로 전용 — 매니지먼트·IC 경로에 주입 금지.**

독립·슈퍼개인 3단계 골격(기간은 상황 맞춤):
- **Phase 1 디지털 무장·틈새 발굴**: AI 활용 포트폴리오 구축 + 사양→성장 산업 리스키링(자격/과정 수료).
- **Phase 2 하이브리드·브랜딩**: 본캐+부캐 사이드 프로젝트 런칭·수익화 + 콘텐츠 발행으로 구독자·첫 외부 수주.
- **Phase 3 슈퍼개인·포트폴리오화**: 수익 파이프라인 다각화(자문·강연·콘텐츠·투자), 급여 외 소득 비중 확대.
- **측정지표는 정량화**: 포폴 유무, 리스키링 수료, 사이드 수익화, 구독자 수, 급여외 소득 비중. (구체 목표 수치·연도는 개인 맞춤/런타임.)

### 6. 직군별 전환 매트릭스(위기요인→생존전략→유망직무)
- **화이트칼라**: 관리자(Administrator)→조율자(Orchestrator). SQL·Python·노코드로 시티즌 디벨로퍼.
- **블루테크(숙련 기능직)**: "데이터를 읽는 기술자" — 센서 데이터 분석·로봇 티칭. (AI가 코드는 짜도 물리 수리·용접은 못 함 → 재평가.)
- **휴먼터치(서비스·헬스케어·교육)**: Human Touch + 디지털 케어 융합.

### 7. 리스키링(산업 환승) + 스킬 매핑 + 갭 분석
- **스킬 매핑 규칙**: 정체 산업 공정기술을 성장 산업 대응 공정으로 "번역"(출신 공정 → 목표 공정 1:1). 예: 디스플레이 박막코팅→배터리 전극코팅, 석유화학 유체역학→배터리 주액.
- **역량 갭 분석**: As-Is(보유) vs To-Be(목표직무 요구)를 **Hard Skill / Soft Skill 2축**으로 분해해 갭 식별·충원.
- 비전공자도 데이터 역량을 더해 인접 직무(기술영업·공급망관리)로 진입 가능.

### 8. 진화 중 직무의 재포지셔닝 + 0-1-3 골격
- 지원 직무가 패러다임 전환 중이면 후보 스토리를 "과거 축"이 아닌 **"새 축(데이터·SW)"에 정렬**. 양손잡이형 인재(도메인 감(感)+데이터 논리(Logic))로 프레이밍.
- **0-1-3년 로드맵 골격**(첨단 직무 재사용): 단기=기초체력(언어·도메인 원리) / 중기=툴 자격증+실습 산출물 / 장기=아키텍처 설계·사양서·벤더 관리.
- 추상적 역량 주장 대신 **증명 가능한 자격증·실습 산출물**로 뒷받침. 정량적 트레이드오프를 설명할 수 있으면 '안정화가'가 아닌 '아키텍트'로 평가.

### 9. JD 역산 매칭(로드맵 연동)
JD 빈출 키워드 추출 → 준비 항목으로 전환 → 각 키워드를 외부교육·자격증·프로젝트 경험에 **1:1 매핑**. 지능형 ATS는 키워드 카운트가 아니라 의미 유사도로 평가하므로 스터핑 금지, 경험을 구체·서사적으로 풀어 역량으로 해석되게 한다. (문서·면접 세부는 각 버킷 참조.)

### 코치용 평가기준
- **즉시전력감**: 준비된 인재·중고신입 선호. 실제 레거시·대용량·현장 경험으로 즉시투입 입증(토이 프로젝트 아님).
- **데이터 논리**: 추상적 말발 아닌 숫자로 증명하는 논리인가(문과 동일 기준).
- **경력 명확성**: 애매한 경력은 감점. 과도한 경력을 신입 지원 시 리스크.
- **융합/희소성**: 두 분야 융합 설명 가능성이 대체불가성·연봉 프리미엄으로 연결.
- **AI 이중무장**: AI 활용력(기본) + 인간 소프트스킬(차별화) 양극단 충족.
- **학습 민첩성 = 최고 스펙**: 특정 프레임워크 사용법보다 새 스택을 빠르게 습득하는 메타학습 능력.

### 로드맵 산출 원칙
- 로드맵은 "결과"가 아니라 "과정"을 담고 3요소(목표·실행·지표)로 구성.
- **수시채용 상시 대응**: 공채·졸업시즌 개념 버리고 역량 완성 시 즉시 지원 가능한 365일 준비 상태.
- **시의성 데이터는 프리즈 금지** — 채용통계·시장규모·연봉·기업 프로그램·특정 툴/모델·예시 정량치는 런타임 JD Browsing으로 재수집(freshness_flags 참조).

아래는 **방법론이 아니라 시의성 데이터**다. 본 모듈에 특정 수치·기업명·연도 라벨을 고정하지 말고,
태스크 실행 시점에 공개 검색(JD Browsing)으로 재수집한다. **프로토콜:** 공개 Phase 0→3 진행 ·
로그인/페이월 도달 시 **즉시 정지 → "인증 필요" 보고**(우회·자격증명 저장 금지) · 못 찾으면 지어내지 말고
'데이터 확보 실패' 명시 + 사용자에게 요청 · 인용은 Truth Tier(S/A/B/C) 라벨 + 조회일 부착.

- **연봉·총보상:** 산업/기업/직급별 연봉·TC 밴드, 성과급 비율(PS/PI/OPI), RSU/사이닝, 이직 인상률 통상치, 세금.
- **기업 고유 정보:** 인재상 키워드·핵심가치·리더십 원칙 문구(Amazon LP, SK SUPEX, 삼성 5품성 등), 신년사·컬처메모, 채용 프로세스명(SKCT/GSAT 등)·AI면접 솔루션 브랜드.
- **JD·직무 요건:** 실시간 JD 원문·URL, 직무별 in-demand 하드스킬/키워드/필수 자격증, 히든 배리어의 구체 요건(규격 버전·사내 코테 범위·본사 영어라운드), 기업 평판.
- **채용시장 통계·트렌드:** ATS/AI 스크리닝 사용률·탈락 원인 비율, 수시채용 비중·중고신입 마지노선, 주니어 채용 증감, 스킬기반 채용률, 산업별 인력수급·성장/소멸 산업 순위.
- **산업·클러스터 지형:** 메가트렌드 목록·상대 비중, 클러스터/팹·라인 지형·투자·조직개편, 정부 제도(히든챔피언·메가클러스터·규제법), 시장 진단 라벨('초양극화'·'피크 코리아' 등).
- **도구·서비스명:** ATS·이력서 진단/첨삭 서비스(Jobscan·Resume Worded 등)의 현재 이용 가능 여부, AI 모델·기술 세부(의미론적 매칭·미세표정 분석 현 수준).
- **예시 정량치:** 생산성 향상률·시간 절감률·직무 반감기 개월수 등 — 예시로만, 근거 수치로 고정 금지.
- **컨설팅/강의 홍보 수치:** 누적 수강생 등 프로모션 데이터는 방법론 근거로 인용 금지.
