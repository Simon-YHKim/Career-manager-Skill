# reference/glossary.md — 초보 용어집 (취린이용 한 줄 풀이)

> 이 스킬·산출물에 나오는 핵심 전문용어를 **아무것도 모르는 초보(취린이)**도 알게 평이한 한국어로 정의.
> SKILL §1 "초보 배려"가 참조. **규칙: 초보로 판단되면 아래 용어를 문서·대화에서 최초 등장 시 괄호로 즉시 풀이**
> (예: "필살기(=면접에서 내세울 대표 성공 경험)"). 경력자에겐 생략 가능. 이건 정의 사전일 뿐, 채점·규율의 원천은 각 reference 문서.

| 용어 | 한 줄 풀이 (초보용) |
|---|---|
| **JD** | **채용공고**(Job Description) — 회사가 "이런 사람 뽑아요"라고 올린 글. 지원 자격·하는 일이 여기 담겨 있어요. |
| **필살기** | 면접·서류에서 내세울 **대표 성공 경험**(유사 경험 + 수치 성과 + 그로부터 얻은 통찰). |
| **메타인지 자가진단** | 내 실력·강약점을 **한 발 떨어져 객관적으로 되짚어 보는 것**("나 지금 어디쯤이지?"). |
| **PR / PR강도** | **자기표현·어필**(회사 홍보 아님). 같은 사실을 약하게(Lv1)→강하게(Lv3, 방어 가능) 표현하는 수위 — 날조는 안 함. |
| **빌살기 / 밉살기** | 빌살기 = 직무와 무관하지만 태도·기초를 보여줄 때 잠깐 빌려 쓰는 경험. 밉살기 = 오히려 감점될 경험(잦은 이직 등) — 방어 논리를 만들거나 뺀다. |
| **무게중심** | 그 회사·직무의 **당락을 가르는 결정적 평가 축**(예: 코딩 실력, 영어, 도메인 지식). 여기에 준비를 몰아준다. |
| **히든 배리어** | 공고엔 크게 안 드러나지만 **실제로 붙고 떨어지는 숨은 요건**(예: 특정 툴 경험, 본사 영어 면접). |
| **Truth Tier (T1·T2·T3)** | 주장의 **근거 등급**. T1 = 문서·수치로 입증됨 / T2 = 사실이나 설명(프레이밍)이 필요 / T3 = 아직 미검증 → **대외 제출엔 안 씀**. |
| **5-D Prism** | 채용공고(JD)를 5개 렌즈(요구역량·회사 속마음·성과지표·공격포인트·조직문화)로 쪼개 나와 맞춰보는 분석틀. |
| **7-Lens** | 지원 회사를 7개 관점(사업·제품·조직·기술·채용시그널·리스크·핏)으로 훑는 기업 분석틀. |
| **STAR-L / C·R·A·R·I** | 경험을 **상황→과제→행동→결과(수치)→배운점** 순서로 정리하는 서술 공식. 자소서·면접 답변의 뼈대. |
| **Tech-to-Biz** | 기술·현장 활동을 **돈·비율·성과 언어로 번역**하는 것(예: "사이클타임 15%↓ → 연 2억 절감"). |
| **ATS** | 회사가 이력서를 1차로 거르는 **자동 스크리닝 시스템**. 표·이미지·특수 서식을 피하고 키워드를 넣어야 통과. |
| **BATNA** | 협상에서 **결렬됐을 때의 나의 대안**(=마지노선). 이게 있어야 무리한 오퍼를 거절할 힘이 생긴다. |
| **Score Caps** | 근거·수치가 없으면 **점수 상한을 두는 규칙**(격려용 고득점 방지). 진단을 냉정하게 유지. |
| **무게중심 매트릭스 / 채용 장벽 매트릭스** | 직무별 핵심 축(예: 코딩·영어·도메인)에 1~5점을 매겨 **어디에 힘을 쏟을지** 찾는 표. |
| **verdict-first** | 리포트 **맨 위에 핵심 판정(점수·등급·개선 Top 3)을 먼저** 보여주고, 근거는 접어두는 방식(경력자용). |

> 여기 없는 용어가 대화에 나오면, 초보에겐 그 자리에서 한 줄로 풀어준다. 이 목록은 확장 가능.
