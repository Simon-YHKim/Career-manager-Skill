# reference/writing-voice.md — 문체 게이트 & 컨설턴트 문체 (P4)

> 목표: AI가 뱅크 기반으로 쓴 글이 **AI 티 없이**, 잘 훈련된 취업 컨설턴트가 쓴 것처럼
> **명확·간결·읽기 쉽게** 나오게 한다. 모든 문서 생성에 자동 적용(게이트). 이력서·자소서는
> 진지 문서 — 이모지·과장 수사 금지. portfolio-builder.md P4가 이 파일을 참조.

## 1. AI-tell 블랙리스트 (걸리면 재작성)

생성 문장이 아래 패턴을 포함하면 게이트에서 막고 고쳐 쓴다.

**국문 상투구 / 공허어:**
- "~을 통해", "~에 있어", "~라고 할 수 있다", "~하는 데 큰 도움이 되었다"
- "다양한", "성공적으로", "적극적으로", "원활한", "효율적으로"(수치 없이), "최선을 다해", "열정을 가지고"
- "뿐만 아니라 ~도", "나아가", "더불어", "이를 바탕으로"(연결어 남발)
- 형용사·부사로 성과를 포장(정량으로 대체).

**영문 AI-tell:**
- "leverage", "utilize"(→ use), "delve", "tapestry", "seamless", "robust", "spearheaded"(남발), "passionate about"
- "In today's fast-paced world", "plays a crucial role", "it's worth noting that"
- 접속 부사 남발: "Moreover, Furthermore, Additionally" 연속.

**구조적 tell:**
- 세 항목 병렬의 기계적 반복("A, B, and C" 리듬이 모든 문장에).
- 문단마다 같은 길이·같은 시작.
- 결론 없는 요약("결론적으로 ~는 중요하다").
- 과한 균형("장점도 있지만 단점도…") — 이력서엔 불필요.

## 2. 컨설턴트 문체 원칙 (지향)

1. **두괄식** — 결론·성과를 문장 맨 앞에. "무엇을 이뤘다" → 그다음 "어떻게".
2. **동사로 시작, 과거형 완결** — 액션 동사 + 정량 결과로 마감.
3. **한 문장 한 메시지** — 접속어로 늘이지 말고 끊는다.
4. **구체 > 추상** — "다양한 프로젝트" ✗ → "결제·정산 2개 파이프라인" ○.
5. **수치로 닫기** — 모든 성과 bullet은 %·₩·건수·배수로 끝나면 강하다(없으면 P2 정량화 대화).
6. **군더더기 제거** — 읽어서 뜻이 안 바뀌면 단어를 뺀다.

## 3. Action Verbs 매핑 (약한 표현 → 강한 동사)

| 약함 | 국문 강 | 영문 강 |
|---|---|---|
| ~을 했다 / 담당했다 | 주도했다, 설계했다, 구축했다 | Led, Designed, Built, Drove |
| 도왔다 / 참여했다 | 개선했다, 최적화했다 | Improved, Optimized, Streamlined |
| 만들었다 | 개발했다, 표준화했다 | Developed, Standardized, Launched |
| 줄였다 | 절감했다(₩), 단축했다(시간) | Reduced, Cut, Accelerated |
| 늘렸다 | 확대했다, 향상시켰다 | Increased, Scaled, Boosted |
| 관리했다 | 리딩했다(N명), 총괄했다 | Managed, Owned, Coordinated |

> 같은 동사를 문서 전체에서 반복하지 않는다(특히 "주도/Led" 남발 주의).

## 4. 진지 문서 규칙 (이력서·자소서·커버레터)

- 이모지·이모티콘·과한 느낌표·구어체 감탄(우와, 정말) **금지**.
- 1인칭 과잉("저는") 최소화 — 행동·결과가 주어.
- 유행어·밈·해시태그 본문 사용 금지(태그는 작업용 뱅크 내부 분류에만).
- 겸양 상투구("부족하지만", "미약하나마") 금지 — 사실과 근거로.

## 5. 낭독 가독성 루프 (STT diff) — 선택

말로 읽기 어려운 문장은 눈으로도 걸린다. STT 입력이 가능할 때:

1. 완성 문단을 사용자에게 **소리 내어 2–3회** 읽게 한다.
2. 음성 → 텍스트(STT)로 받는다.
3. 원문과 **diff**: 사용자가 무의식적으로 (a) 바꿔 읽은 단어, (b) 끊어 읽은 지점, (c) 다시 읽은(더듬은) 구간을 찾는다.
4. 그 구간 = 실제로 어색하거나 숨 가쁜 문장 → **우선 개선**(짧게 쪼개거나 어순 교정).
5. STT 입력이 없으면 이 루프는 **스킵**(문체 게이트만 적용).

**면접 응용 (태스크 ④ 음성 모의면접):** 동일 STT 파이프를 모의면접 답변에 적용 → **전달력 실측** 채점: 필러("음·어") 빈도 · 답변 길이/시간 · 두괄식 여부 · 더듬음·재시작 · 명료도 → evaluation.md §5.1 '전달력' 축을 텍스트 추정이 아니라 실측으로. 글 개선용 루프가 곧 구술 리허설 도구가 된다.

## 6. 자동 게이트 절차 (생성 시마다)

```
초안 생성 → [1] AI-tell 블랙리스트 스캔 → 걸린 문장 재작성
        → [2] 진지문서면 이모지/감탄 제거 확인
        → [3] Action Verb 매핑으로 약한 동사 교체
        → [4] 각 성과 bullet 수치 마감 확인(없으면 P2 정량화 플래그)
        → [5] (STT 있으면) 낭독 루프 → 최종
```
게이트는 **출력 모드와 무관하게** 항상 적용. 제출용은 여기에 더해 증빙·메모·해시태그를 숨긴다.
